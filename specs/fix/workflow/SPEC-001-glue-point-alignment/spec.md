# Feature Specification: HMI 胶点预览锚定分布一致性

**Feature Branch**: `fix/workflow/SPEC-001-glue-point-alignment`  
**Created**: 2026-03-29  
**Status**: Draft  
**Input**: User description: "修复 HMI 胶点预览与规划触发点分布，使胶点按每条几何线段首尾双锚定并在目标 3.0 mm 间距附近均匀分布，确保几何交点与共享顶点严格共点，预览仅消费权威插补结果上的显式位置触发，禁止回退到运动轨迹、距离推导或固定间隔采样，并为短线段建立豁免与正确校验。"

## Clarifications

### Session 2026-03-29

- Q: 旧格式结果处理策略是什么？ → A: 只接受当前规则下的权威结果；旧格式或历史结果一律明确失败，不做自动转换。
- Q: 如何判定“角点/交点完全重合”？ → A: 以几何坐标重合为准，允许极小数值舍入误差，但不允许形成可分辨的近邻点。
- Q: 预览失败是否阻止继续执行当前计划？ → A: 是；权威计划胶点预览失败时，当前计划不得继续执行。
- Q: 预览与执行准备的一致性口径是什么？ → A: 预览与执行准备必须消费同一份权威出胶结果。
- Q: 短线段例外如何判定？ → A: 只要在线段首尾双锚定前提下，均匀分布得到的实际间距落出 2.7 mm 至 3.3 mm，就判定为短线段例外。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 执行前确认角点与交点 (Priority: P1)

作为在执行前做最终判断的操作者或评审人，我希望 HMI 展示的计划胶点预览在每个几何端点、角点和共享交点都与目标几何完全重合，这样我才能相信预览结果可以作为执行前的门禁依据。

**Why this priority**: 角点和共享顶点是否准确是本需求的核心价值。如果关键交点都不能按当前验收口径重合，预览在执行前就不具备决策价值。

**Independent Test**: 使用包含方框、对角线和共享顶点的参考图形生成预览，只检查几何交点和共享顶点是否全部被单一胶点准确命中，即可独立证明最核心价值。

**Acceptance Scenarios**:

1. **Given** 用户加载了包含多个直线段和共享顶点的有效几何图形，**When** 系统生成计划胶点预览，**Then** 每个应出胶的几何端点和共享顶点都必须出现一个与该顶点坐标完全重合的胶点。
2. **Given** 多条线段在同一物理顶点相交，**When** 用户查看该顶点附近的预览结果，**Then** 系统必须展示一个公共胶点，而不是多个彼此接近但不重合的点簇。

---

### User Story 2 - 均匀查看非整倍数边长的布点结果 (Priority: P2)

作为负责工艺可读性和可解释性的工艺工程师，我希望系统在每条几何线段上先固定首尾两点，再在中间均匀分布胶点，并允许点距围绕 3.0 mm 做有限浮动，这样我可以得到既满足几何锚定、又具有稳定视觉与工艺一致性的分布结果。

**Why this priority**: 即使角点命中，如果中间布点仍沿整条路径按固定步距推进，线段长度不是目标间距整数倍时仍会出现末端漂移和局部密度不均。

**Independent Test**: 选取边长不是 3.0 mm 整数倍的矩形或折线，只验证每条线段的首尾锚定、段内均匀性和常规段点距窗口，即可独立证明本故事成立。

**Acceptance Scenarios**:

1. **Given** 某条应出胶线段的长度不是目标间距的整数倍，**When** 系统生成该线段的胶点预览，**Then** 系统必须保留该线段的首尾两点，并在两点之间生成均匀分布的中间胶点，使实际点距尽可能接近目标间距。
2. **Given** 某条应出胶线段很短，无法在保留首尾两点的同时继续满足常规点距窗口，**When** 系统生成预览，**Then** 系统必须优先保留首尾锚点，并将该线段按短线段例外规则处理，而不是牺牲端点重合来满足常规窗口。

---

### User Story 3 - 拒绝非权威来源的计划预览 (Priority: P3)

作为需要判断预览是否可信的开发、测试或评审人员，我希望计划胶点预览只来自当前计划中与执行准备共用的那一份权威出胶结果，并在权威来源缺失时明确失败，这样我不会把推导、回退或历史残留结果误认为当前可执行预览。

**Why this priority**: 即使几何分布规则正确，只要系统还能在权威来源缺失时偷偷回退到其他来源，预览就仍然可能和真实执行意图脱节。

**Independent Test**: 人为构造权威来源缺失、但其他回退来源存在的场景，只验证系统是否拒绝生成预览并给出明确失败理由，即可独立证明该故事。

**Acceptance Scenarios**:

1. **Given** 当前计划缺少权威的显式位置触发结果，**When** 用户请求生成计划胶点预览，**Then** 系统必须明确返回失败，而不是改用其他轨迹、距离列表或固定间隔推导预览点。
2. **Given** 当前计划存在可消费且与执行准备共用的权威显式位置触发结果，**When** 用户生成计划胶点预览，**Then** 系统必须基于该权威结果生成预览，并且同一输入在重复生成时给出一致的胶点坐标。
3. **Given** 当前计划的权威计划胶点预览生成失败或校验失败，**When** 用户尝试继续执行当前计划，**Then** 系统必须阻止该计划继续执行，直到预览重新生成并通过当前规则校验。

---

### Edge Cases

- 几何图形包含共享顶点，例如闭合轮廓与对角线共用同一顶点。
- 线段长度不是目标间距的整数倍，但两端都必须保留锚点。
- 在线段首尾双锚定前提下，均匀分布得到的实际间距落出 2.7 mm 至 3.3 mm 的线段需要按短线段例外处理。
- 几何输入包含零长度段、重复段或仅由数值噪声形成的无效段。
- 当前计划缺少权威触发结果，但系统仍持有历史预览或其他可推导轨迹。
- 多个彼此不相连的轮廓同时存在，系统不能把轮廓之间的跳跃误判为相邻点距异常。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须仅基于当前计划的权威显式位置触发结果生成计划胶点预览。
- **FR-002**: 当权威显式位置触发结果缺失、为空或不可追溯到当前计划时，系统必须拒绝生成计划胶点预览并返回明确失败原因。
- **FR-003**: 系统必须禁止以其他轨迹来源、距离推导结果、固定间隔采样结果或历史残留结果替代当前计划的权威显式位置触发结果。
- **FR-004**: 对每条应出胶的几何线段，系统必须保留线段首点和尾点作为胶点锚点。
- **FR-005**: 系统必须在每条应出胶线段的首尾锚点之间均匀分布中间胶点，使实际点距尽可能接近目标间距 3.0 mm。
- **FR-006**: 对于常规长度线段，系统必须将生成后的实际胶点圆心间距控制在目标间距的默认容差窗口内，即 2.7 mm 至 3.3 mm。
- **FR-007**: 当线段在首尾双锚定前提下，按均匀分布计算得到的实际点距落出 2.7 mm 至 3.3 mm 常规容差窗口时，系统必须优先保留首尾锚点，并将该线段作为短线段例外单独处理。
- **FR-008**: 系统必须保证每个应出胶的几何端点、角点和共享顶点都由一个与目标几何重合的胶点表示；允许仅由数值表示导致的极小舍入误差，但不得形成可分辨的近邻点或替代公共顶点的点簇。
- **FR-009**: 系统必须防止同一共享顶点被多个彼此接近但不重合的胶点重复表示。
- **FR-010**: 系统必须对不同轮廓、不同源线段或其他几何分组分别评估胶点间距，不能把互不相邻的路径跳跃当成同一连续段的点距异常。
- **FR-011**: 系统必须在相同输入图形和相同计划前提下生成稳定、可重复的胶点坐标，不得因重复预览而出现坐标漂移或点数波动。
- **FR-012**: 当几何输入存在零长度段、重复点或其他无法形成可信布点结果的异常时，系统必须明确处理这些异常，并避免将无效段转化为误导性胶点。
- **FR-013**: 系统必须提供足够的来源信息，使评审人能够判断当前预览是否属于当前图形、当前计划和当前执行前检查上下文。
- **FR-014**: 系统必须确保计划胶点预览与对应的执行准备结果消费同一份权威出胶结果；对于同一图形和同一计划，预览中呈现的应出胶位置不得与执行准备结果分离、冲突或分别独立生成。
- **FR-015**: 当系统为了满足几何锚定和均匀分布而调整权威出胶位置结果时，系统必须保持原有路径顺序、应出胶区间边界和连续出胶语义不变，不得引入额外的出胶段、遗漏既有出胶段，或改变既定的执行先后关系。
- **FR-016**: 对于不满足当前权威来源规则的历史结果、旧格式结果或缺少必要来源信息的结果，系统必须明确拒绝将其作为计划胶点预览依据；禁止自动转换，也禁止静默沿用旧规则生成预览。
- **FR-017**: 当同一图形在等价的计划条件下被重复规划、重新打开或重新进入执行前检查时，系统必须保持胶点数量、共享顶点命中关系和各线段布点结果的一致性，使评审人能够得到稳定的判断依据。
- **FR-018**: 当权威计划胶点预览无法生成、来源不合规或未通过当前规则校验时，系统必须阻止当前计划继续执行，直到预览结果重新满足当前规则。

### Key Entities *(include if feature involves data)*

- **应出胶几何线段**: 需要参与点胶布点的单条直线段，是首尾锚定和段内均匀分布的基本单位。
- **共享顶点**: 被两条或多条线段共同使用的物理顶点，必须在预览中表现为一个公共胶点坐标。
- **计划胶点触发结果**: 当前计划中被认定为权威来源的出胶位置结果，用于支撑计划胶点预览。
- **计划胶点预览**: 面向操作者和评审人的执行前可视化结果，展示本次计划应如何在几何路径上布置胶点。
- **点距校验分组**: 用于复核胶点间距的真实几何分组，防止将不相邻路径错误拼接后得出污染性统计结果。

### Assumptions

- 本需求范围限定在执行前的计划胶点预览，不覆盖真实点胶后的物理成型效果。
- 目标间距默认为 3.0 mm，常规容差窗口默认为 ±10%，即 2.7 mm 至 3.3 mm。
- 当常规容差窗口与端点重合要求冲突时，以端点和共享顶点的几何正确性为更高优先级。
- 输入图形中的应出胶路径已经被识别为业务上允许参与预览的几何对象。

### Dependencies

- 必须存在可追溯到当前计划、且可被预览与执行准备共同消费的权威出胶结果。
- 必须存在可用于判定端点、角点和共享顶点的有效几何输入。
- 必须存在能区分当前计划结果与历史残留结果的来源识别信息。

### Out of Scope

- 真实执行时的运动控制、胶量控制和成品工艺质量验证。
- HMI 视觉样式、配色或非功能性渲染表现优化。
- 弧线、曲线或其他非直线几何对象的布点规则扩展。
- 与本次计划无关的历史数据清理策略和通用数据迁移方案。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在参考验收样例中，100% 应出胶的几何端点、角点和共享顶点都由与目标几何重合的单一胶点表示；允许仅由数值表示导致的极小舍入误差，但不允许出现可分辨的近邻点或点簇替代公共顶点。
- **SC-002**: 在参考验收样例中，100% 未触发短线段例外规则的线段，其实际胶点圆心间距落在 2.7 mm 至 3.3 mm 窗口内；100% 触发该规则的线段都保留首尾锚点并被正确归类为例外处理。
- **SC-003**: 在权威触发结果缺失的测试场景中，100% 的计划胶点预览请求都以明确失败结束，且不会产生基于其他来源推导的替代预览结果。
- **SC-004**: 在包含多个轮廓或共享顶点的参考样例中，100% 的点距复核结果都按真实几何分组输出，不会把跨轮廓跳跃误判为相邻胶点点距。
- **SC-005**: 对于有效参考样例，评审人员能够在 2 分钟内完成从打开预览到判断“可作为执行前依据/不可作为执行前依据”的检查流程。
- **SC-006**: 在权威计划胶点预览生成失败、来源不合规或校验失败的测试场景中，100% 的当前计划执行请求都会被阻止，且不会以告警代替门禁。
