# Feature Specification: 胶点全局均布与真值链一致性

**Feature Branch**: `fix/workflow/SPEC-002-global-glue-spacing`  
**Created**: 2026-03-29  
**Status**: Draft  
**Input**: User description: "修复胶点分布问题，将胶点规划改为按连续工艺路径全局均布，闭环支持相位优化，并统一预览、校验与执行对同一权威胶点结果的消费，避免按段锚定导致的局部堆点、疏密跳变和起点偏置。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 连续路径均匀布点 (Priority: P1)

作为在执行前审核胶点预览的工艺工程师或操作者，我希望系统把每条连续点胶路径作为一个整体来布点，而不是受内部几何分段方式影响，这样我才能把预览中的疏密分布当作可信的执行依据。

**Why this priority**: 当前最直接的业务风险是同一条真实路径会因为内部几何切分不同而出现局部堆点、局部变疏或闭环偏置，导致预览失去决策价值。

**Independent Test**: 使用同一条真实轮廓的两份等价输入，一份采用少量长段，一份采用多段细分，只验证胶点数量、顺序和整体分布是否保持一致，即可独立证明该故事。

**Acceptance Scenarios**:

1. **Given** 同一条连续点胶路径被表达为两种几何上等价但分段数量不同的输入，**When** 系统分别生成胶点预览，**Then** 两次结果必须保持相同的胶点数量、相同的路径顺序，并给出等价的整体分布。
2. **Given** 一条开放的连续点胶路径长度不是目标间距的整数倍，**When** 系统生成胶点预览，**Then** 系统必须在保留路径边界的前提下沿整条路径均匀分布胶点，而不是在局部出现明显的密集区或稀疏区。
3. **Given** 一条闭合点胶路径没有业务上必须固定的起始点，**When** 系统生成胶点预览，**Then** 结果不得表现出由固定起点造成的缝口堆点、缝口大间隙或整圈明显偏置。

---

### User Story 2 - 预览、校验与执行同源 (Priority: P2)

作为需要根据预览决定是否允许继续执行当前计划的评审人，我希望预览、间距校验和执行准备共享同一份权威胶点结果，这样我看到的内容才能真实代表系统即将执行的内容。

**Why this priority**: 即使几何分布本身正确，只要预览、校验和执行来自不同点集，就仍然会出现“看起来通过、实际不可执行”或“显示结果与执行语义脱节”的风险。

**Independent Test**: 构造一组权威胶点结果存在、另一组权威结果缺失或失效的场景，只验证系统是否分别给出一致预览和明确失败，即可独立证明该故事。

**Acceptance Scenarios**:

1. **Given** 当前计划存在可追溯且有效的权威胶点结果，**When** 用户打开胶点预览并执行校验，**Then** 预览展示、间距结论和执行准备必须基于同一份结果。
2. **Given** 当前计划缺少有效的权威胶点结果，**When** 用户请求生成胶点预览或尝试继续执行，**Then** 系统必须明确失败并阻止继续执行，而不是改用推导结果、历史结果或显示专用结果替代。
3. **Given** 同一份有效计划被重复打开、重新校验或重新进入执行前检查，**When** 用户多次查看胶点预览，**Then** 系统必须返回一致的胶点数量、顺序和校验结论。

---

### User Story 3 - 闭环与曲线场景可解释 (Priority: P3)

作为负责评估复杂轮廓可制造性的工艺工程师，我希望闭合轮廓、圆弧和平滑曲线上的胶点分布遵循真实可见路径，并在无法满足常规间距窗口时给出可解释的例外信息，这样我才能判断当前结果是合理例外还是规划异常。

**Why this priority**: 实际图形并不只包含直线；如果闭环和曲线仍然受固定缝口、控制骨架或异常输入污染，用户看到的结果仍然不可信。

**Independent Test**: 使用包含闭合轮廓、圆弧、平滑曲线和超短局部路径的参考样例，只验证布点是否沿真实路径分布、闭环无固定缝口偏置且例外场景有明确说明，即可独立证明该故事。

**Acceptance Scenarios**:

1. **Given** 一条包含圆弧或平滑曲线的连续点胶路径，**When** 系统生成胶点预览，**Then** 胶点必须落在最终可见的点胶路径上，并保持与直线场景同等的均匀性目标。
2. **Given** 某段连续路径在保留必要边界或强约束锚点后无法满足常规间距窗口，**When** 系统生成胶点预览，**Then** 系统必须保留几何正确性并明确标记该结果为例外，而不是通过隐藏、漂移或伪均匀方式掩盖问题。
3. **Given** 几何输入包含零长度段、重复点或断裂路径定义，**When** 系统生成胶点预览，**Then** 系统必须安全处理这些异常，并避免展示误导性的胶点结果。

---

### Edge Cases

- 同一条真实路径被导入为不同的几何分段组合，但业务含义完全等价。
- 闭合轮廓没有天然起点，且多个顶点都不应被默认当作强制锚点。
- 连续路径中存在极短局部区域，无法在保留必要边界后继续满足常规间距窗口。
- 同一工件同时包含多个互不相连的轮廓，系统不能把轮廓之间的跳跃误判为相邻点距。
- 路径包含直线、圆弧和平滑曲线的混合几何。
- 几何输入包含零长度段、重复点、重叠点或仅由数值噪声形成的无效段。
- 当前计划缺少权威胶点结果，但系统仍持有历史预览或其他可推导轨迹。
- 同一有效计划被重复打开或重新规划时，系统不能出现胶点数量漂移、顺序改变或缝口位置跳变。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须将每条连续点胶路径作为一个整体来确定胶点布局，而不是按内部几何分段各自独立布点。
- **FR-002**: 对于几何上等价、仅内部切分方式不同的输入，系统必须给出相同的胶点数量、相同的路径顺序和等价的整体分布结果。
- **FR-003**: 对于开放路径，系统必须在保留该路径业务边界的前提下，沿整条路径均匀分布胶点，使相邻胶点间距尽可能接近目标间距。
- **FR-004**: 对于闭合路径，系统必须在没有业务强制起点时避免依赖固定几何起点生成胶点布局。
- **FR-005**: 对于相同的闭合路径和相同的计划输入，系统必须稳定选择同一套胶点布局，不得因重复生成而出现缝口位置跳变或整圈偏置变化。
- **FR-006**: 系统必须保留业务上必需的路径边界、强约束锚点、出胶顺序和连续出胶语义。
- **FR-007**: 系统不得把仅用于几何表达的内部切分顶点、导入伪顶点或其他非业务约束点默认当作必须落胶的锚点。
- **FR-008**: 系统必须仅在同一条连续路径或同一独立轮廓内部评估相邻胶点间距，不得把不连续跳跃误判为相邻点距。
- **FR-009**: 当某条路径或局部区域在保留必要边界或强约束锚点后无法满足常规间距窗口时，系统必须优先保持几何正确性，并将该结果标记为例外且提供明确原因。
- **FR-010**: 系统必须使胶点分布遵循最终可见的点胶几何路径，包括直线、圆弧和平滑曲线，而不是遵循仅用于中间表达的控制骨架或辅助结构。
- **FR-011**: 对于相同输入，系统必须生成稳定、可重复的胶点数量、顺序和坐标结果。
- **FR-012**: 系统必须使用同一份权威胶点结果作为预览展示、间距校验和执行准备的共同来源。
- **FR-013**: 当权威胶点结果缺失、过期、不一致或未通过校验时，系统必须拒绝将其作为可执行预览，并阻止当前计划继续执行，直到结果恢复有效。
- **FR-014**: 系统必须确保用户在预览中看到的胶点集合就是当前计划最终认可的胶点布局本身，而不是单独重建、近似还原或显示专用的替代点集。
- **FR-015**: 系统必须安全识别并处理零长度段、重复点、断裂路径或其他无效几何，避免生成误导性的胶点结果。
- **FR-016**: 系统必须提供足够的来源与状态信息，使评审人能够判断当前预览属于当前图形、当前计划和当前执行前检查上下文。
- **FR-017**: 当存在多个互不相连的轮廓时，系统必须分别生成、分别校验并分别保持各轮廓的胶点顺序与结果，不得跨轮廓混算。
- **FR-018**: 当用户在不改变输入条件的情况下重复打开、重复校验或重新进入执行前检查时，系统必须返回一致的预览结论和一致的通过/失败判定。

### Key Entities *(include if feature involves data)*

- **连续点胶路径**: 一条需要连续评估胶点分布的业务路径，可由多段几何元素组成，但在布点时被视为一个整体。
- **闭合点胶路径**: 起点与终点连成闭环、没有天然业务起点的连续点胶路径。
- **强约束锚点**: 业务上必须保留的边界点或关键点，系统需要围绕它们生成最终胶点布局。
- **权威胶点结果**: 当前计划唯一被认可为可用于预览、校验和执行准备的胶点布局结果。
- **胶点间距例外**: 在保留几何正确性和必要边界后，无法满足常规间距窗口但仍被系统明确解释和标记的结果。
- **执行前检查上下文**: 用户查看预览、执行校验并决定是否继续执行当前计划时所依赖的当前图形、当前计划和当前状态信息集合。

### Assumptions

- 目标胶点间距默认为 3.0 mm，常规间距窗口默认为 2.7 mm 至 3.3 mm，除非后续需求明确调整。
- 本需求关注执行前的胶点布局正确性与可解释性，不直接覆盖真实出胶后的物理成型质量。
- 需要出胶的路径范围已经在上游业务流程中被识别完成。
- 若同一工件包含多个互不相连的轮廓，每个轮廓都被视为独立的连续点胶路径单元。

### Dependencies

- 必须存在可绑定到当前计划的权威胶点结果来源。
- 必须存在可识别连续路径、闭合路径和独立轮廓关系的有效几何输入。
- 必须存在执行前检查流程对预览结果和校验结论的消费入口。

### Out of Scope

- 实际执行时的运动控制细节、阀控时序和胶量物理成型质量。
- HMI 的视觉风格、配色、图标或其他纯展示样式优化。
- 人工手动编辑单个胶点位置的交互设计。
- 与当前计划无关的历史数据迁移、旧结果自动转换或通用清洗策略。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在开放路径和闭合路径的参考验收样例中，100% 未触发例外规则的相邻胶点间距都落在 2.7 mm 至 3.3 mm 窗口内；100% 触发例外规则的结果都被明确标记并附带原因。
- **SC-002**: 对于几何上等价、仅内部切分方式不同的参考样例，100% 的结果都保持相同的胶点数量和相同的路径顺序，对应胶点位置偏差不超过 0.1 mm。
- **SC-003**: 在闭合路径参考样例中，100% 的重复生成结果都保持相同的胶点数量、相同的整体分布和相同的通过/失败判定，不出现缝口位置跳变或局部堆点位置漂移。
- **SC-004**: 在权威胶点结果缺失、过期、不一致或校验失败的测试场景中，100% 的预览请求都会以明确失败结束，且 100% 的后续执行请求都会被阻止。
- **SC-005**: 在包含直线、圆弧和平滑曲线的参考样例中，100% 的胶点都位于目标可见点胶路径上，且不会出现仅在预览中存在、但不属于最终认可胶点布局的额外点集。
- **SC-006**: 在标准参考样例上，评审人员能够在 2 分钟内完成从打开预览到做出“允许继续执行/禁止继续执行”判断的检查流程。
