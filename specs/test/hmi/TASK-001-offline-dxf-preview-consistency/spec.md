# Feature Specification: HMI 在线点胶轨迹预览一致性验证

**Feature Branch**: `test/hmi/TASK-001-offline-dxf-preview-consistency`
**Created**: 2026-03-26
**Status**: Draft
**Input**: User description: "验证 HMI 在在线模式下基于真实 DXF 文件生成点胶轨迹预览，并确认预览结果在业务语义上与上位机准备下发到运动控制卡的数据一致；表示格式可以不同；机台已经准备就绪。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 在线查看真实轨迹 (Priority: P1)

作为需要在执行前做判断的操作者或测试评审人，我希望在 HMI 已进入在线就绪且机台已经准备好的前提下加载真实 DXF 并看到点胶轨迹预览，这样我可以在真正准备执行前判断当前路径是否可信。

**Why this priority**: 这是本需求的核心价值。如果在线就绪状态下无法看到可信预览，后续一致性验证和执行前判断都无从成立。

**Independent Test**: 在 HMI 在线就绪且机台准备好的状态下加载一个经过认可的真实 DXF，生成预览并完成人工查看；整个过程应体现真实执行前的在线业务上下文。

**Acceptance Scenarios**:

1. **Given** HMI 已进入在线就绪状态且机台已经准备好，并且用户提供了真实、有效的 DXF 文件，**When** 用户触发轨迹预览，**Then** 系统应展示可供人工判断的点胶轨迹预览。
2. **Given** 同一真实 DXF 在相同任务上下文和相同在线就绪前提下被重复预览，**When** 用户多次执行在线预览，**Then** 系统应生成可复现、可比较的预览结果。

---

### User Story 2 - 判断预览是否与执行准备一致 (Priority: P2)

作为测试评审人，我希望确认 HMI 显示的预览与上位机当前准备下发到运动控制卡的数据在业务语义上保持一致，这样预览才能真正承担执行前门禁作用，而不是仅供展示的独立结果。

**Why this priority**: 预览如果与执行准备结果脱节，会直接削弱执行前确认的有效性，并放大误判风险。

**Independent Test**: 在同一 DXF 和同一任务上下文下，对照预览结果与对应的执行准备记录，验证几何路径、路径顺序和点胶相关运动语义不存在无法解释的差异，即使表示格式不同。

**Acceptance Scenarios**:

1. **Given** 预览结果和对应的执行准备记录来自同一 DXF 与同一任务上下文，**When** 评审人进行比对，**Then** 两者在几何路径、路径顺序和点胶相关运动语义上应保持一致，即使表示格式不同。
2. **Given** 预览结果无法证明与当前执行准备记录对应，**When** 评审人尝试将其作为真实轨迹预览证据，**Then** 该结果不得被判定为通过。

---

### User Story 3 - 识别非权威预览结果 (Priority: P3)

作为需要签收测试结果的人员，我希望系统能明确区分权威在线预览和模拟、历史残留或其他非当前任务结果，这样我不会把错误结果误当成真实轨迹证据。

**Why this priority**: 该能力决定了验收证据是否可信，尤其是在存在 mock、占位数据或历史残留结果的情况下。

**Independent Test**: 分别提供权威预览和非权威预览场景，验证系统能清楚标识差异，并阻止后者被当成真实轨迹验证通过的依据。

**Acceptance Scenarios**:

1. **Given** 当前显示的预览来自模拟、占位或其他非权威来源，**When** 用户查看预览结果，**Then** 系统应明确提示该结果不能作为真实轨迹验收依据。
2. **Given** 当前预览结果已过期、与当前 DXF 不匹配、缺乏来源追溯信息或不属于当前在线就绪上下文，**When** 用户尝试继续使用该结果完成验收，**Then** 系统应阻止其被记录为真实轨迹预览通过。

---

### Edge Cases

- HMI 已启动，但尚未进入在线就绪状态或机台准备状态失效。
- 真实 DXF 文件可被打开，但其中没有可用于点胶预览的有效路径。
- 预览画面存在内容，但内容来自历史结果、其他任务上下文或当前 DXF 之外的数据。
- 预览几何外观看似合理，但路径顺序或点胶相关运动语义与执行准备结果不一致。
- 输入 DXF 文件损坏、不完整或包含当前流程无法支持的内容，导致无法形成可靠预览。
- 预览结果无法证明其来源是否权威时，不得以“看起来正常”为理由默认通过。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须允许用户在 HMI 在线就绪且机台已经准备好的状态下生成点胶轨迹预览。
- **FR-002**: 任何用于验证真实轨迹预览能力的测试案例都必须以真实 DXF 文件作为输入，不得以手工构造、占位或模拟几何数据替代。
- **FR-003**: 系统必须确保预览结果与对应的执行准备数据来自同一 DXF 和同一任务上下文。
- **FR-004**: 系统必须保证预览结果与上位机当前准备下发到运动控制卡的数据在几何路径、路径顺序和点胶相关运动语义上保持一致。
- **FR-005**: 系统必须允许预览结果与控制卡下发数据采用不同表示形式，但两者在验收口径上的业务语义必须等价。
- **FR-006**: 系统必须明确区分权威在线预览结果与模拟、占位、历史残留或其他非权威结果。
- **FR-007**: 系统必须阻止模拟、占位、过期、不可追溯、来源不匹配或不属于当前在线就绪上下文的预览结果被记录为真实轨迹预览通过。
- **FR-008**: 用户必须能够通过预览结果及其关联信息判断当前结果是否属于目标 DXF、目标任务和当前在线就绪上下文。
- **FR-009**: 当 DXF 无法形成可用点胶预览，或当前未满足在线就绪前提时，系统必须给出明确失败结果，而不是展示误导性内容或复用旧结果。
- **FR-010**: 系统必须保留足够的关联信息，使预览结论能够追溯到被评审的 DXF、相应的执行准备结果以及当前在线就绪上下文。

### Key Entities *(include if feature involves data)*

- **真实 DXF 文件**: 用于触发在线预览的工程图纸输入，应代表业务上可被认可的实际图纸数据。
- **在线预览结果**: HMI 在在线就绪状态下展示给用户的点胶轨迹可视化结果及其摘要信息，用于执行前人工判断。
- **执行准备数据**: 上位机为后续控制卡下发所准备的权威业务结果，是本需求下一致性判断的参照对象。
- **预览来源状态**: 用于表明当前预览是否来自权威业务结果，还是来自模拟、占位、过期或不可追溯来源的判定信息。
- **验证证据记录**: 用于记录本次预览结论、输入来源及关联上下文的审计材料。

### Assumptions

- 本需求默认覆盖单工件、单任务、标准执行前检查流程。
- 本需求关注执行前可判断性，不承诺胶量、铺展或其他工艺结果一致性。
- HMI 在线就绪且机台准备好被视为进入真实预览检查的前置状态。

### Dependencies

- 必须存在可被业务接受的真实 DXF 来源，用于构成正式验收输入。
- 必须能够获得与预览结果对应的执行准备结果，用于做一致性比对。
- 必须能够识别当前预览结果是否属于权威来源，以及是否对应当前任务和当前在线就绪上下文。

### Out of Scope

- 机台从未就绪到就绪的建链、调机或恢复流程。
- 点胶工艺效果、胶量分布、成品质量或物理精度验收。
- DXF 编辑、修改或多任务调度流程。
- 异常中断恢复、复杂现场故障处理或多工件批处理场景。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 100% 计划用于真实轨迹验收的测试案例都能在 HMI 在线就绪且机台准备好的状态下，以真实 DXF 文件完成预览发起与查看。
- **SC-002**: 100% 被判定为通过的案例都不存在预览结果与对应执行准备数据之间在几何路径、路径顺序和点胶相关运动语义上的无法解释差异。
- **SC-003**: 100% 被接收为真实轨迹证据的预览结果都能追溯到被评审的 DXF、匹配的执行准备结果以及当前在线就绪上下文。
- **SC-004**: 100% 属于模拟、占位、过期、来源不匹配、不可追溯或不满足在线就绪前提的预览结果都不能被记录为真实轨迹预览通过。
- **SC-005**: 对于有效 DXF 案例，评审人能够在 3 分钟内完成从加载图纸到做出预览通过/不通过判断的在线检查流程。
