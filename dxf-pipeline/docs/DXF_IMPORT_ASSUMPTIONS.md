# DXF 导入假设

## 当前输入约束

- `dxf-pipeline` 当前面向固定输入：`DXF R12`。
- 输入几何默认是二维平面数据，处理链路以 `mm` 为内部单位输出。
- 上游若产生高版本 DXF 特性，需在进入本仓库前完成降级、炸开或离散化。

## R12 对实现的含义

- 优先支持 `LINE`、`ARC`、`CIRCLE`、旧式 `POLYLINE` / `VERTEX`、`POINT`。
- 不应把 `LWPOLYLINE`、`SPLINE`、`ELLIPSE`、`INSERT`、复杂对象代理当作主输入路径。
- 如代码中保留了对高版本实体的兼容分支，应视为兜底逻辑，而不是主流程。

## 简化方向

- 可把旧式 `POLYLINE` 作为主轮廓结构，减少多实体分支。
- 可降低对样条离散、椭圆采样、块引用展开等路径的测试与维护优先级。
- 可把 DXF 单位和二维约束前移为输入契约，减少运行时推断与容错分支。
