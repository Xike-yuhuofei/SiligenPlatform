# 开发规范

## 分层原则

- `geometry` 负责几何原语与基础运算。
- `path` 负责路径结构化与优化。
- `trajectory` 负责时序采样、约束与报告。
- `contracts` 负责版本化输入输出定义。
- `adapters` 负责对接外部消费方格式。

## 编码原则

- 算法代码与 CLI 壳层分离。
- 输出结构的任何扩展都要考虑版本兼容性。
- 性能优化变更要保留最小基准对照。

## DXF 导入边界

- 当前仓库以 `DXF R12` 作为固定输入前提，不把高版本 DXF 兼容性作为默认目标。
- 新增 DXF 处理逻辑时，优先验证 `LINE`、`ARC`、`CIRCLE`、旧式 `POLYLINE` / `VERTEX` 的正确性。
- 若需要支持 `SPLINE`、`ELLIPSE`、`LWPOLYLINE`、`INSERT` 等非 R12 核心实体，应先明确这是上游预处理责任还是本仓库责任。
