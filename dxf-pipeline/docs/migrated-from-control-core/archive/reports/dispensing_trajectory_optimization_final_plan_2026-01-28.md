# 点胶轨迹算法优化最终方案（C++20 可用版）

**日期**：2026-01-28  
**修订**：2026-01-28（依据评估补充 PUR 工艺细节、UDP 抖动缓解与时间线调整）  
**状态**：最终方案 / 可实施（含补充细节）  
**适用范围**：龙门式 XY+Z（2–8 轴），MultiCard + 以太网 UDP，针阀（时间/位置触发），PUR 热熔胶，高粘度，2D 轨迹（直线/圆弧/样条）

---

## 1. 目标与约束（冻结版）

### 1.1 目标
- **终极目标**：优化点胶轨迹算法，提升线宽均匀性与拐角质量。
- **精度目标**：线宽均匀性 5–10%（精度优先于速度）。
- **节拍目标**：低速（>30s/板），允许牺牲速度换取精度。
- **落地周期**：长期演进（4–6 个月），允许架构级重构。

### 1.2 关键约束
- **架构约束**：六边形架构，领域层禁止动态内存（需转移大对象至基础设施层）。
- **实时性**：非实时总线，依赖软件补偿。
- **硬件**：无编码器反馈；时间域触发为主；CMP 位置触发需硬件验证。
- **技术栈**：C++20 可用（此前 C++17）。

---

## 2. 总体方案（最终选型）

**主线组合（精度优先 + 低速场景最优）**：
1) **几何预处理（拐角平滑）**
2) **全局速度前瞻（双向扫描）**
3) **Jerk 约束插补（Ruckig 为主，自研兜底）**
4) **等距触发点计算（曲线段二分搜索）**
5) **触发策略优化（subsegmented + 残差补偿 + 匀速段点胶）**
6) **流量-速度同步 + 触发安全边界**
7) **PUR 工艺前馈（温度/压力/粘度补偿曲线）**

**关键决策**：
- **Ruckig 可引入**（C++20 允许），但**仅替代“速度曲线引擎”**，不能替代“几何平滑与前瞻速度规划”。
- **时间触发仍是主路径**（无编码器反馈），采用 subsegmented 改善加减速堆胶；如硬件验证通过，可升级为“位置触发”。
- **曲率限速安全系数**：对 PUR 高粘度建议 `v_max = 0.8 * sqrt(a_max * R)` 作为初始保守策略。

## 3. 系统架构与模块划分

### 3.1 端到端流水线
```
DXF 解析
  → 路径链接/去断点
  → 几何平滑（圆弧/样条过渡）
  → 速度前瞻（曲率 + 加速度约束）
  → 轨迹插补（Ruckig 或 S 曲线）
  → 触发规划（等距触发 / subsegmented / 匀速点胶 / 安全边界）
  → 输出到 MultiCard
```

### 3.2 领域层（Domain）
- **GeometryBlender**：拐角圆弧/样条过渡生成
- **SpeedPlanner**：双向扫描 + 曲率限速
- **TriggerPlanner**：等距触发点计算 + subsegmented 触发 + 残差补偿 + 安全边界
- **TrajectoryAssembler**：从 MotionSegment 生成可执行轨迹

> 领域层仅存轻量结构，避免动态分配；大缓存由基础设施层提供。

### 3.3 基础设施层（Infrastructure）
- **DXFFileParsingAdapter**：DXF 解析与初步段生成
- **RuckigVelocityProfileAdapter**：实现 IVelocityProfilePort
- **MultiCardAdapter / ValveAdapter**：运动与触发执行
- **Visualization / CSV Trace**：轨迹与速度可观测性

## 4. 算法细节（可落地）

### 4.1 几何平滑（优先圆弧过渡）
- 直线-直线/直线-圆弧拐角处插入过渡圆弧。
- 可配置最大偏移 `corner_max_deviation_mm`，优先 0.05–0.1 mm。
- `corner_smoothing_radius` 建议 0.3–0.5 mm 起步（精密点胶更稳妥）。
- 若对曲率连续性有更高要求，可升级样条过渡（B-Spline/Bezier）。

### 4.2 速度前瞻（双向扫描）
- **前向扫描**：确保加速不超限。
- **反向扫描**：确保减速可达。
- **曲率限速**：`v_max = sqrt(a_max * R)`；PUR 建议乘安全系数 0.8。

### 4.3 Jerk 约束插补
- **Ruckig 作为主引擎**：生成 jerk-limited 时间最优速度曲线。
- **自研 S 曲线兜底**：保持 5/7 段 S 曲线接口可切换。

### 4.4 等距触发点计算（二分搜索）
- **目标**：在曲线/样条段上获得等距触发点，解决“曲线胶点不均匀”。
- **方法**：对每段建立“弧长 s → 参数 t/点索引”的映射，使用二分搜索求解触发点。
- **输出**：
  - **时间触发**：弧长 → 时间（通过速度曲线积分）→ 生成 `interval_ms` 序列或 subsegmented 配置；
  - **位置触发**：直接输出等距触发位置数组（MC_CmpBufData）。

### 4.5 触发策略（时间触发）
- **默认策略**：subsegmented（N=8），加/减速段细分。  
- **残差补偿**：累计距离残差，抵消 `interval_ms` 量化误差。  
- **匀速段点胶**：可配置“加减速禁胶”策略，以最小化堆胶。

### 4.6 触发安全边界
- 若 `interval_ms < duration_ms + valve_response_ms` ⇒ **必须阻止或降速**。
- 软边界：`duration_ms + valve_response_ms ≤ interval_ms < duration_ms + valve_response_ms + margin_ms` ⇒ 警告。

### 4.7 位置触发（可选升级）
- 若硬件支持 MC_CmpBufData + Profile Position（无需编码器），可切换为位置触发；需验证 MultiCard 的稳定性与延迟。

### 4.8 PUR 工艺补偿（新增细化）
- **起胶补偿**：开阀提前量（ms）+ 预出胶点；避免起笔断胶。  
- **收胶补偿**：关阀滞后 + 回吸（若硬件支持），降低拉丝与拖尾。  
- **拐角胶量调整**：拐角减速时适当减小脉宽或触发密度，避免堆胶。  
- **温度/粘度曲线**：建立温度 → 粘度 → 速度上限的经验模型，作为速度/触发修正因子。

### 4.9 UDP 抖动缓解
- **插补点预缓冲**：在适配层建立 50–100 点的预缓冲队列。  
- **通信异常检测与重试**：超时重传与降速兜底。  
- **速度突变检测与平滑**：对相邻段速度变化做限幅，避免抖动放大。

## 5. 配置与数据结构调整（建议）

### 5.1 配置参数扩展
- **InterpolationConfig**：
  - `corner_max_deviation_mm`
  - `corner_min_radius_mm`
  - `blend_strategy`（Arc/Spline）
  - `curvature_speed_factor`

- **DispensingConfig**：
  - `dispensing_strategy`（baseline/segmented/subsegmented）
  - `subsegment_count`
  - `dispense_only_cruise`
  - `valve_response_ms`
  - `visual_margin_ms`
  - `min_interval_ms`
  - `open_comp_ms`（起胶提前量）
  - `close_comp_ms`（收胶滞后量）
  - `retract_enabled`（回吸开关）
  - `corner_pulse_scale`（拐角胶量缩放）
  - `prebuffer_points`（预缓冲点数）

### 5.2 默认参数建议
- `time_step = 0.001s`
- `corner_smoothing_radius = 0.5mm`（精密点胶更稳妥）
- `corner_max_deviation_mm = 0.1mm`
- `look_ahead_points = 10`
- `curvature_speed_factor = 0.8`（PUR 高粘度保守系数）
- `dispensing_velocity = 20–50 mm/s`（PUR 保守）
- `trigger_interval = 1.0mm`（可在 0.5–1.0mm 内调参）
- `subsegment_count = 8`
- `prebuffer_points = 50–100`（UDP 抖动缓解）

## 6. 分阶段实施计划（4–6 个月）

### Phase 0：基线验证（1–2 周）
- 建立线宽测量标准、采样频率、显微镜/视觉方法
- 基线数据采集：线宽 CV、拐角堆胶、触发间距统计

### Phase 1：架构与核心能力（5–7 周）
- 等距触发点计算（二分搜索）
- subsegmented 触发 + 残差补偿
- 路径链接 + 拐角平滑（圆弧过渡）
- 速度前瞻（双向扫描）
- 安全边界与日志

### Phase 2：插补引擎升级（4–6 周）
- 引入 Ruckig（C++20）
- 保留自研 S 曲线兜底
- 速度前瞻 → Ruckig 入参对接

### Phase 3：工艺前馈与收敛（6–8 周）
- PUR 温度/压力/粘度补偿曲线
- 起胶/收胶补偿与回吸策略
- 拐角胶量调整与线宽收敛
- 参数标定与最终验收

## 7. 验收标准（强制）

- **线宽均匀性**：CV ≤ 10% 或 (max-min)/mean ≤ 10%。
- **拐角质量**：拐角线宽偏差 ≤ +10%。
- **堆胶改善**：Phase 1 ≥ 30%，Phase 1.5 ≥ 50%。
- **触发间距**：匀速段误差 ≤ 10%，加减速段显著优于 baseline。
- **安全边界**：违反 interval/duration/response 时必须拒绝执行或强制降速。
- **稳定性**：连续 20 次任务触发数量偏差 ≤ 1%，无漏触发/连触发。

---

## 8. 主要风险与缓解

- **时间域触发 vs 空间均匀**：通过 subsegmented、匀速点胶、残差补偿降低误差。
- **UDP 抖动**：插补点预缓冲（50–100 点）、通信超时重试、速度突变检测与平滑。
- **PUR 拉丝/断胶**：起胶提前、收胶滞后与回吸策略，必要时降低拐角速度与脉宽。
- **无编码器反馈**：优先验证 Profile Position CMP 位置触发可行性。
- **MultiCard 固件兼容**：提前与供应商确认 CMP/缓冲能力并做硬件验证。
- **领域层内存约束**：使用基础设施层预分配缓冲并传递视图。
