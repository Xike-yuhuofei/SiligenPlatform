# 排胶 UseCase 方案设计

**日期**: 2026-01-19  
**角色视角**: 点胶机工程师  
**目标**: 形成可落地的排胶（排气/清胶）流程编排与参数定义，确保安全、稳定、可追溯

## 背景

排胶用于清除管路气泡、恢复点胶稳定性或换胶后预排胶。典型场景包括：
- 开机预排（消除停机期间产生的空洞）
- 换胶/换针头后的排气
- 维护后恢复点胶稳定性
- 产线短暂停机后的胶路恢复

## 目标

1. 提供标准化的排胶流程（可自动控制供胶阀与点胶阀）
2. 支持连续开阀与按住排胶（释放即停止）
3. 具备安全互锁与异常回退能力
4. 输出可追溯的排胶记录（参数、时长、结果）

## 非目标

- 不在该用例内执行运动定位（由外部 MoveToPositionUseCase 或人工完成）
- 不在该用例内实现流量闭环控制（需传感器与称重系统支持）

## 用例定义

**名称**: PurgeDispenserUseCase  
**命名空间**: `Siligen::Application::UseCases::Dispensing::Valve::Dispenser`  
**职责**: 排胶流程编排（供胶阀开/稳/关 + 点胶阀连续开阀）

## 触发与使用场景

- 手动触发：HMI/物理按键（参考 `docs/library/06_reference.md` 的排胶键定义），按住排胶，释放即停止
- 自动触发：开机流程、换胶流程、异常恢复流程中的固定步骤
- CLI/HMI 实现：`dispenser purge`（支持 `--wait-purge-key` 监听 X2，按下开始，释放停止）

## 前置条件与安全互锁

1. 硬件连接正常、急停未触发
2. 系统状态为 IDLE/READY/PAUSED（不得处于 RUNNING）
3. 喷嘴处于排胶位置且具备回收容器
4. 供胶压力与温度达到工艺要求
5. 门禁/光幕/气压等互锁信号满足安全条件

## 输入参数建议

| 参数 | 说明 | 建议默认值 |
|---|---|---|
| supply_stabilization_ms | 供胶阀打开后稳定等待 | 500 ms |
| wait_for_completion | 是否阻塞等待排胶结束 | true |
| timeout_ms | 最大等待/安全时长 | 30000 ms |

补充参数：
- `manage_supply`: 是否由用例自动管理供胶阀（默认 true）
- `poll_interval_ms`: 等待完成时的轮询间隔（默认 50 ms）

> 架构规范落地：供胶稳压时间由领域层 `SupplyStabilizationPolicy` 统一解析，
> Application 仅传递可选覆盖值；覆盖范围 0-5000ms，0 表示使用配置默认值；
> Infrastructure 仅负责阀门 I/O，不包含时序规则。

> 说明：排胶采用连续开阀，`wait_for_completion=false` 时由调用方负责停止点胶阀与关闭供胶阀。

## 执行流程（MVP）

1. 校验请求参数（范围与组合逻辑）
2. 校验硬件连接与安全互锁
3. 打开供胶阀并等待稳定
4. 打开点胶阀（连续开阀排胶）
5. 轮询状态直到停止/释放/超时/异常
6. 关闭点胶阀与供胶阀并返回结果

## 异常处理策略

- 供胶阀打开失败：立即返回错误
- 点胶阀启动失败：关闭供胶阀并返回错误
- 排胶超时：关闭点胶阀、关闭供胶阀并返回超时错误
- 运行中进入暂停：判定为异常，返回状态错误
- 运行中被停止：视为结束，关闭供胶阀并返回结果

## 日志与追溯

建议记录：
- 排胶参数（连续开阀时长/是否按住排胶）
- 启动/结束时间
- 是否完成、是否超时
- 错误码与错误信息

## 验证建议

1. 单元测试：参数校验、超时逻辑、异常回退路径
2. 集成测试：供胶阀/点胶阀联动时序
3. 硬件验证：排胶量稳定性、飞溅控制、针头回滴情况
