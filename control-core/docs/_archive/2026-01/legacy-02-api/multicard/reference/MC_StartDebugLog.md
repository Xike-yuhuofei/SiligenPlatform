# MC_StartDebugLog 函数教程

## 函数原型

```c
int MC_StartDebugLog(short nLogType)
```

## 参数说明

| 参数名 | 类型 | 说明 |
|--------|------|------|
| nLogType | short | 日志类型。0：关闭调试日志；非0：开启调试日志 |

## 返回值

| 返回值 | 说明 |
|--------|------|
| 0 | 成功 |
| 非0 | 失败，错误代码 |

## 实际代码示例

### 示例1：开启调试日志（连接前）
```cpp
// 在连接控制卡前开启调试日志
int iRes = 0;

g_MultiCard.MC_StartDebugLog(0);  // 0表示关闭调试日志
// 一个板卡就声明1个对象，多个板卡就声明多个对象，各个对象之间互不干涉
// 5个参数依次为板卡号、PC端IP地址、PC端端口号，板卡端IP地址，板卡端口号。
// 通常建议不同板卡IP地址要不同，端口使用0让系统自动分配

iRes = g_MultiCard.MC_Open(1,"192.168.0.200",0,"192.168.0.1",0);
```

### 示例2：通过按钮控制调试日志
```cpp
// 通过复选框控制调试日志的开启/关闭
void CDemoVCDlg::OnBnClickedCheckPrintSpeed()
{
    g_MultiCard.MC_StartDebugLog(0);  // 传递0来控制调试输出
}
```

## 参数映射表

| 使用场景 | nLogType | 说明 |
|----------|----------|------|
| 关闭调试日志 | 0 | 禁用调试信息输出 |
| 开启调试日志 | 1 | 启用调试信息输出 |
| 切换状态 | 0/1 | 通过复选框或菜单切换状态 |

## 关键说明

1. **调试功能**：
   - 输出详细的通信和执行信息
   - 帮助开发者诊断问题
   - 显示API调用的执行结果
   - 记录运动控制的详细过程

2. **使用时机**：
   - **开发调试**：在开发和测试阶段开启
   - **问题诊断**：遇到异常时开启查看详细信息
   - **性能优化**：分析运动执行的时序和状态
   - **生产环境**：建议关闭以提高性能

3. **日志内容**：
   - 网络通信状态
   - API调用结果
   - 运动执行状态
   - 错误和警告信息

4. **注意事项**：
   - 开启调试日志会影响系统性能
   - 大量的调试输出可能影响实时性
   - 生产环境中建议关闭
   - 与MC_GetCardMessage()配合使用获取详细错误信息

## 函数区别

- **MC_StartDebugLog() vs MC_GetCardMessage()**: MC_StartDebugLog()控制调试输出开关；MC_GetCardMessage()获取卡的具体消息
- **开启 vs 关闭**: 参数为0时关闭，非0时开启
- **调试 vs 生产**: 开发时开启调试，生产时关闭以提高性能

---

**调试建议**: 在开发阶段保持调试日志开启，有助于快速定位问题。程序稳定后关闭以提高运行效率。
