# DXF文件驱动点胶功能使用指南

## 概述

本功能允许用户上传DXF文件，自动解析图形路径，生成运动轨迹，并协调电机与电磁阀实现精准点胶。

### 核心能力

- **DXF解析**: 支持AutoCAD DXF格式文件（LINE和ARC实体）
- **路径规划**: 自动生成符合运动学约束的平滑轨迹
- **阀门协调**: 时间触发或位置触发(CMP)模式
- **可视化预览**: SVG实时预览轨迹和触发点
- **参数配置**: 灵活的速度、加速度、触发间隔等参数

## 支持的DXF实体

| 实体类型 | 说明 | 限制 |
|---------|------|------|
| LINE | 直线段 | 无 |
| ARC | 圆弧段 | 仅支持二维平面圆弧 |

**注意**: 不支持POLYLINE、SPLINE、3D实体等复杂图元。

## 操作流程

### 1. 上传DXF文件

#### 步骤

1. 导航到 **"DXF路径规划"** 页面
2. 点击 **"选择DXF文件"** 按钮
3. 选择符合要求的 `.dxf` 文件
4. 点击 **"上传文件"** 按钮

#### 文件要求

- **格式**: AutoCAD 2000或更高版本
- **扩展名**: `.dxf`
- **大小**: < 10 MB
- **内容**: 仅包含LINE和ARC实体
- **路径**: 图形应连续（无断点）

#### 验证

上传成功后显示：
- ✅ 绿色提示框：`"文件上传成功"`
- 显示文件名和大小
- 文件路径保存到服务器 `/tmp/uploads/`

**错误处理**:
- 无效文件类型：`"仅支持.dxf文件"`
- 解析失败：`"DXF文件解析失败 - 错误详情"`

---

### 2. 配置轨迹参数

上传成功后，配置以下参数：

| 参数 | 说明 | 范围 | 推荐值 | 影响 |
|------|------|------|--------|------|
| **最大速度** | 运动最大线速度 | 1-1000 mm/s | 100-200 mm/s | 执行速度 |
| **最大加速度** | 运动最大加速度 | 1-10000 mm/s² | 1000-3000 mm/s² | 加速性能 |
| **最大加加速度** | Jerk限制（S曲线） | 1-100000 mm/s³ | 10000 mm/s³ | 平滑度 |
| **时间步长** | 插补周期 | 0.0001-0.1 s | 0.001 s | 轨迹精度 |
| **圆弧容差** | 弧线拟合精度 | 0.001-1.0 mm | 0.01 mm | 弧线质量 |
| **点胶间隔** | 触发点中心距(点径1.0mm+边缘间隔1.0mm) | 0.1-100 mm | 2.0 mm | 点胶密度 |
| **触发脉宽** | 阀门脉冲宽度 | 10-5000 μs | 100 μs | 阀门开启时间 |

#### 点胶策略与补偿参数（2026-01新增）

| 参数 | 说明 | 推荐 | 影响 |
|------|------|------|------|
| **strategy** | 触发策略：baseline/segmented/subsegmented/cruise_only | subsegmented | 触发密度与堆胶控制 |
| **subsegment_count** | 加减速细分段数量 | 8 | 曲线/拐角均匀性 |
| **dispense_only_cruise** | 仅在匀速段点胶 | false | 减少起收胶堆胶 |
| **corner_smoothing_radius** | 拐角平滑半径(mm) | 0.3-0.5 | 拐角几何过渡 |
| **corner_max_deviation_mm** | 拐角最大偏移(mm) | 0.1 | 几何精度约束 |
| **curvature_speed_factor** | 曲率限速系数 | 0.8 | 曲线段速度安全系数 |
| **open_comp_ms** | 起胶提前补偿(ms) | 0-5 | 减少起胶拖尾 |
| **close_comp_ms** | 收胶滞后补偿(ms) | 0-5 | 减少断胶 |
| **retract_enabled** | 回吸开关 | false | 缓解拉丝 |
| **corner_pulse_scale** | 拐角脉宽倍率 | 1.0-1.5 | 抑制拐角堆胶 |

#### 触发安全边界（建议开启）

| 参数 | 说明 | 推荐 | 影响 |
|------|------|------|------|
| **valve_response_ms** | 阀响应时间(ms) | 5 | 最小触发间隔计算 |
| **visual_margin_ms** | 视觉裕量(ms) | 10 | 触发安全冗余 |
| **min_interval_ms** | 最小触发间隔(ms) | 10 | 过密触发自动降级 |

#### 路径优化

勾选 **"启用路径优化"** 可自动：
- 对DXF图元重新排序，减少空行程
- 从指定起点（默认(0,0)）开始优化
- 适用于非连续路径的DXF文件

**注意**: 如果DXF路径已经是有序的，无需启用优化。

### 3. 路径规划

#### 步骤

1. 确认参数配置正确
2. 点击 **"路径规划"** 按钮
3. 等待后端计算（通常 1-5 秒，取决于DXF复杂度）

#### CLI 快速验证（无需UI）

> 适用于开发/回归验证：直接生成预览与导出数据。

```bash
# 路径规划 + 速度曲线导出
siligen_cli dxf-plan --file=examples/i_love_you.dxf --no-interactive --velocity-trace
```

输出文件（与 DXF 同目录或 logs 目录）：
- `*_preview.html`：轨迹预览
- `*_glue_points.csv`：触发点导出（触发间距复核）
- `*_velocity_trace.csv`：速度曲线导出（默认 50ms 采样）

默认参数来源：
- 未指定 `--max-vel/--max-acc` 时，读取 `config/machine_config.ini` 的 `[Machine] max_speed/max_acceleration`
  （若缺失则回退 `[Hardware] max_velocity_mm_s/max_acceleration_mm_s2`）。
- 触发点复核阈值可通过请求参数覆盖：`spacing_tol_ratio/spacing_min_mm/spacing_max_mm`（仅告警，不阻断）。

#### 结果显示

规划成功后显示：

| 显示项 | 说明 | 示例 |
|-------|------|------|
| **轨迹段数量** | LINE和ARC段总数 | 42 |
| **总路径长度** | 所有段长度之和 | 125.6 mm |
| **预计执行时间** | 按当前速度估算 | 12.5 s |
| **触发点数量** | 阀门触发次数 | 125 |
| **SVG预览** | 可视化轨迹图 | 实时渲染 |

#### SVG预览功能

- ✅ **方向箭头**: 显示运动方向
- ✅ **起点/终点标记**: 绿色起点，红色终点
- ✅ **触发点标记**: 橙色圆点（按间隔分布）
- ✅ **缩放/平移**: 鼠标滚轮缩放，拖拽平移

**错误处理**:
- 参数超出范围：`"参数验证失败 - 详细信息"`
- DXF文件不存在：`"DXF文件未找到"`
- 规划失败：`"路径规划失败 - 错误详情"`

### 4. 执行点胶

#### 前置条件

1. ✅ **硬件已连接**
   - 运动控制卡已上电
   - 阀门控制卡已连接
   - 后端状态显示 **"已连接"**

2. ✅ **轴参数已配置**
   - 脉冲当量（mm/pulse）
   - 方向设置
   - 速度/加速度限制

3. ✅ **阀门已配置**
   - IO端口映射
   - 触发模式（时间/CMP）
   - 脉冲宽度

#### 执行步骤

1. 导航到 **"DXF点胶执行"** 页面
2. 确认轨迹信息显示正确
3. 点击 **"开始点胶"** 按钮
4. 观察实时进度：
   - 进度条百分比
   - 已完成点胶次数
   - 实时状态（**运行中**）
5. 等待执行完成

#### 速度偏离保护（开环时间触发）

当使用时间触发（非硬件位置触发）时，系统可按规划速度与实时速度进行偏离监控：
- `velocity_guard_enabled`：是否启用
- `velocity_guard_ratio`：相对偏差阈值
- `velocity_guard_abs_mm_s`：绝对偏差阈值
- `velocity_guard_min_expected_mm_s`：最小期望速度阈值
- `velocity_guard_grace_ms`：启动宽限期
- `velocity_guard_interval_ms`：采样间隔
- `velocity_guard_max_consecutive`：连续违规次数
- `velocity_guard_stop_on_violation`：超限是否停止

默认行为：启用监控，仅告警不停止（可按工艺调整）。

#### 停止操作

如需中止执行：
- 点击 **"停止点胶"** 按钮
- 系统立即停止运动和阀门输出
- 显示当前进度和状态

**注意**: 停止后无法恢复，需重新执行。

## 常见问题

### Q: 上传DXF文件后显示"解析失败"？

**可能原因**:

1. **DXF格式不兼容**
   - 检查DXF文件版本（需AutoCAD 2000+）
   - 尝试用AutoCAD重新保存为R2000格式

2. **不支持的实体类型**
   - 确保文件仅包含LINE和ARC实体
   - 用CAD软件炸开（EXPLODE）复杂图元
   - 删除POLYLINE、SPLINE、文字、标注等

3. **路径不连续**
   - 检查图元端点是否相连
   - 使用CAD软件的JOIN命令连接断点

4. **文件损坏**
   - 尝试用CAD软件修复（RECOVER命令）
   - 重新导出DXF文件

### Q: 规划时提示"参数超出范围"？

**检查清单**:

| 参数 | 检查项 |
|------|--------|
| 速度 | 必须 > 0，建议 ≤ 1000 mm/s |
| 加速度 | 必须 > 0，建议 ≤ 10000 mm/s² |
| 时间步长 | 0.0001-0.1 s，建议 0.001 s |
| 圆弧容差 | 0.001-1.0 mm |
| 点胶间隔 | 必须 > 0，建议 ≤ 10 mm |

**常见错误**:
- ❌ 速度设为0
- ❌ 加速度为负数
- ❌ 时间步长过小（<0.0001）导致计算量过大

### Q: 执行时电机不动或阀门不触发？

**硬件检查清单**:

1. **运动控制卡**
   - ✅ 已上电（电源指示灯亮）
   - ✅ 已连接（后端状态显示"已连接"）
   - ✅ 轴参数已配置（脉冲当量、方向）
   - ✅ 伺服驱动器已使能

2. **阀门控制**
   - ✅ IO端口映射正确
   - ✅ 触发模式设置（时间/CMP）
   - ✅ 阀门电源已打开
   - ✅ 电磁阀线圈无故障

3. **后端日志**
   ```bash
   # 查看实时日志
   tail -f logs/motion-controller.log

   # 搜索错误信息
   grep ERROR logs/motion-controller.log
   ```

**常见错误**:
- `Card not found`: 检查运动控制卡连接
- `Axis not configured`: 配置轴参数
- `IO timeout`: 检查阀门控制卡连接

### Q: 点胶效果不均匀？

**调整建议**:

| 症状 | 可能原因 | 解决方案 |
|------|---------|---------|
| 间隔不均匀 | 点胶间隔过大 | 减小 `点胶间隔` 参数（如 0.5 mm） |
| 漏胶 | 触发脉宽过大 | 减小 `触发脉宽` 参数（如 50 μs） |
| 出胶不足 | 触发脉宽过小 | 增大 `触发脉宽` 参数（如 200 μs） |
| 起点/终点多胶 | 加减速过快 | 减小 `最大加速度` |

### Q: 规划时间过长？

**优化方案**:

1. **简化DXF文件**
   - 删除不必要的图元
   - 合并共线段
   - 减小圆弧数量

2. **调整参数**
   - 增大 `时间步长`（0.005 s）
   - 增大 `圆弧容差`（0.1 mm）

3. **启用路径优化**
   - 勾选 `启用路径优化`
   - 设置合理的起点坐标

## 技术细节

### DXF解析原理

> 说明：该节内容已过时。系统已迁移为 Python 预处理链路（ezdxf → .pb → Python 轨迹），不再使用旧 C++ 解析实现。
```
center:      (cx, cy)
radius:      r
start_angle: θ1 (弧度)
end_angle:   θ2 (弧度)
```

### 轨迹生成算法

#### 1. 直线段插补

```cpp
// 线性插补（Linear Interpolation）
for (t = 0; t <= 1; t += time_step) {
    point = start + (end - start) * t
}
```

#### 2. 圆弧段插补

```cpp
// 圆弧插补（Circular Interpolation）
for (θ = start_angle; θ <= end_angle; θ += dθ) {
    x = center_x + radius * cos(θ)
    y = center_y + radius * sin(θ)
}
```

#### 3. 速度规划

**S曲线加减速**:
```
acceleration(t) = jerk * t          // 加速度段
velocity(t) = v0 + a*t              // 速度积分
position(t) = p0 + v*t + 0.5*a*t²   // 位置积分
```

**约束条件**:
- |velocity| ≤ max_velocity
- |acceleration| ≤ max_acceleration
- |jerk| ≤ max_jerk

#### 4. 时间同步

固定周期插补（`time_step` = 1ms）:
```
trajectory_points[0] -> t = 0.000s
trajectory_points[1] -> t = 0.001s
trajectory_points[2] -> t = 0.002s
...
```

### 阀门触发机制

#### 时间触发模式

按固定时间间隔触发阀门脉冲：

```cpp
// 伪代码
if (current_time - last_trigger_time >= dispensing_interval) {
    TriggerValve(pulse_width);
    last_trigger_time = current_time;
}
```

**优点**: 简单可靠，与速度无关
**缺点**: 高速时可能导致触发点不均匀

#### 位置触发模式（CMP）

运动轴到达指定位置时自动触发：

```cpp
// 伪代码
SetupCMPTrigger(axis, position, pulse_width);
EnableCMPOutput();
```

**优点**: 位置精确，适合高速应用
**缺点**: 需要硬件支持CMP功能

### SVG预览生成

系统将轨迹转换为SVG格式：

```html
<svg viewBox="0 0 800 600">
  <!-- LINE段 -->
  <line x1="100" y1="100" x2="200" y2="150"
        stroke="blue" stroke-width="2"
        marker-end="url(#arrow)" />

  <!-- ARC段 -->
  <path d="M 150 150 A 50 50 0 0 1 200 200"
        stroke="green" stroke-width="2" fill="none" />

  <!-- 触发点 -->
  <circle cx="150" cy="150" r="3" fill="orange" />
</svg>
```

**变换矩阵**:
- 缩放：将物理坐标(mm)映射到SVG像素
- 平移：将轨迹居中显示
- 旋转：可选的旋转校正
