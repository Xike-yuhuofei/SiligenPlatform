## 使用指南

### Checklist使用原则
1. **按顺序执行**: 严格按照清单顺序检查，不要跳过步骤
2. **逐项确认**: 每个检查项都要明确确认完成
3. **记录结果**: 记录每项检查的实际结果，便于问题追踪
4. **失败处理**: 任何检查项失败时，参考对应故障排除指南

### Checklist维护
- 版本控制: 每次更新都应更新版本号和日期
- 实际验证: 新版本发布前必须经过实际测试验证
- 用户反馈: 收集用户使用反馈，持续改进检查项

### 自定义扩展
如需创建新的checklist：
1. 参考现有checklist结构和格式
2. 包含充分的背景信息和上下文
3. 提供具体的验证方法和标准
4. 包含故障排除指南和技术支持信息
