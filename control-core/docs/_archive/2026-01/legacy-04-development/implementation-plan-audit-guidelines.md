## 二、审计目标

创建一个可重用的审计模板，用于评估未来所有实施计划的：
1. **完整性** - 计划结构是否完整
2. **合规性** - 是否符合架构和安全规则
3. **可行性** - 技术实施是否可行
4. **风险管理** - 风险识别和缓解是否充分

### Step 2：架构合规性审计

#### 2.1 六边形架构验证
- [ ] **依赖方向正确**：Infrastructure → Application → Domain（单向）
- [ ] **Domain层纯洁性**：零技术依赖，无硬件/IO/线程
- [ ] **端口访问**：通过 `I*Port` 接口访问 Domain 层
- [ ] **无反向依赖**：Domain 层不依赖 Infrastructure
- [ ] **适配器单一职责**：每个适配器实现一个端口接口

#### 2.2 命名空间合规性
- [ ] **根命名空间**: `Siligen`
- [ ] **分层命名空间**：
  - `src/domain/**` → `Siligen::Domain::*`
  - `src/application/**` → `Siligen::Application::*`
  - `src/infrastructure/**` → `Siligen::Infrastructure::*`
  - `src/adapters/**` → `Siligen::Adapters::*`
  - `src/shared/**` → `Siligen::Shared::*`
- [ ] **头文件纯度**：禁止 `using namespace` 指令

#### 2.3 领域层约束检查
- [ ] **无动态内存**：禁止 `new`、`delete`、`malloc`
- [ ] **STL容器限制**：禁止 `std::vector`、`std::map`（motion模块除外）
- [ ] **STL例外处理**：如需使用，必须添加 `CLAUDE_SUPPRESS` 抑制注释
- [ ] **noexcept标记**：公共API必须标记 `noexcept`
- [ ] **无IO/线程**：禁止 `std::cout`、`std::thread`

### Step 3：工业安全约束审计

#### 3.1 实时性约束
- [ ] **控制循环**：无动态内存分配
- [ ] **日志记录**：控制循环中禁止日志
- [ ] **STL容器**：控制循环中禁止使用
- [ ] **线程安全**：使用 `std::scoped_lock`

#### 3.2 紧急停止合规性
- [ ] **无阻塞操作**：禁止可能阻塞的函数调用
- [ ] **无条件执行**：紧急停止路径无条件判断
- [ ] **无日志记录**：紧急停止路径不记录日志
- [ ] **响应时间**：保证10ms内返回

#### 3.3 硬件隔离
- [ ] **Domain层纯洁**：无直接硬件头文件包含
- [ ] **端口访问**：通过 `IHardwareController_port` 访问硬件
- [ ] **适配器单一性**：每个硬件适配器实现单一端口
