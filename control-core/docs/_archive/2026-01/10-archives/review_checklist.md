# 代码审查检查清单

## 架构与分层
- [ ] 命名空间符合规范（`Siligen::Layer::*`）
- [ ] Domain 层无技术依赖
- [ ] 适配器实现单一端口接口
- [ ] 通过构造函数注入依赖
- [ ] 接口与实现分离（`.h` 仅声明，`.cpp` 仅实现）
- [ ] 头文件自包含且 include 最小化，禁止循环依赖

## 错误处理与异常
- [ ] 使用 `Result<T>` 模式处理错误
- [ ] Domain 公共 API 标记 `noexcept`
- [ ] 异常边界清晰：跨层接口不抛异常或统一转 `Result<T>`

## 资源与所有权
- [ ] 资源管理遵循 RAII，禁止手动 `new/delete`
- [ ] 统一使用智能指针（`std::unique_ptr`/`std::shared_ptr`），禁止裸指针所有权

## 并发与性能
- [ ] 线程安全路径有明确锁策略与必要注释
- [ ] 不可变对象标记 `const`，接口按值/引用传递符合性能规范

## 日志与可观测性
- [ ] 使用 `SILIGEN_LOG_*` 宏记录日志
- [ ] 日志包含上下文（请求 ID/关键参数），禁止记录敏感信息

## 业务规则与用例
- [ ] Use Case 包含请求验证
- [ ] 输入输出边界校验完整（长度、空值、范围、格式）

## 代码规范与可维护性
- [ ] 删除拷贝构造和赋值运算符
- [ ] 通过 clang-format 格式化
- [ ] 添加必要注释和文档
- [ ] 配置项集中管理（禁止散落 magic numbers/strings）

## 测试与变更管理
- [ ] 单元测试覆盖关键用例与错误分支
- [ ] ABI 兼容性检查（对外接口变更需记录在 `CHANGELOG.md`）
