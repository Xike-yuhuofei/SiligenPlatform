## 架构原则

本系统采用六边形架构（Ports & Adapters模式），核心理念：
- 业务逻辑与技术实现分离
- 依赖方向：适配器 → 应用 → 领域 ← 基础设施
- 领域层定义接口，基础设施层实现接口
