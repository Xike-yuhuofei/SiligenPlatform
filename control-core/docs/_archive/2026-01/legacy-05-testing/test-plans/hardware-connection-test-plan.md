硬件控制卡连接测试与验证方案

**测试目标**
- 确认以太网 UDP 连接稳定可靠，基本 IO 功能与串口通信可用，满足性能与可靠性指标。

**测试环境**
- 硬件：控制卡（24V DC 供电）、急停按钮（常闭）、继电器与电磁阀（24V DC）。
- 上位机：Windows 主机，独立网卡，IP `192.168.0.200`，子网掩码 `255.255.255.0`。
- 网络：`Cat5e/Cat6` 直通线；`RJ45` 连接。
- 软件与 SDK：`MultiCard.dll/.lib/.h`（静态链接），配置文件 `machine_config.ini`，抓包工具 `Wireshark`。
- 本仓库 Web UI：后端 API `3010` 端口、前端开发 `5173` 端口，用于可视化验证。

**基础功能测试用例**
- 连接测试（UDP）
  - 前置：完成物理连接与 IP 设置；`MC_StartDebugLog(0)` 开启日志。
  - 步骤：调用 `MC_Open(1, "192.168.0.200", 0, "192.168.0.1", 0)`；记录返回码与耗时。
  - 预期：返回 `0`；耗时 < `2s`；Wireshark 可见 UDP 握手/心跳（示例端口）。
  - 变体：端口固定 `60000`；多卡连接 `192.168.0.2/0.3`（卡号 `2/3`）。
- 急停读取测试（X1）
  - 步骤：调用 `MC_GetDiRaw` 读取 DI；正常态高电平；按下急停变低电平。
  - 预期：状态位 `bit0` 由 `1→0`；响应 < `100ms`；应用层标记急停触发。
- 输出控制测试（Y2）
  - 电平方式：`MC_SetDoBit(..., y2_bit_index, 高/低)`，并 `MC_GetDo` 验证；继电器吸合/释放。
  - 脉冲方式：`MC_CmpPluse(2, 2, 0, 2000, ...)`；监测继电器与电磁阀响应。
  - 预期：输出变化与机械动作一致；错误返回 `-6` 为连接问题，`1` 为参数问题。
- 串口/UART 通信测试
  - 配置：`MC_UartConfig(uart=0, 115200, 8, 无校验, 1)`；双方参数一致。
  - 发送：`MC_SendEthToUartString(uart=0, "TEST", 4)`；接收端抓取与校验。
  - 预期：发送成功，实际吞吐与误码率满足指标。
- Web UI 验证用例
  - 后端/前端服务启动；在测试面板使用默认 IP `192.168.0.1` 执行“连接”“开始测试”。
  - 预期：连接状态更新；接收 `test.progress`（0→100%）与 `test.completed` 事件。

**性能测试指标**
- 连接建立时间：平均 < `1s`，90 分位 < `2s`。
- 连接稳定性：连续连接-断开 100 次，失败率 < `1%`。
- UDP 报文丢包率：在轻负载下 < `0.1%`，在中负载下 < `1%`（以 Wireshark 或应用统计）。
- IO 响应时间：急停检测 < `100ms`；Y2 电平切换至继电器吸合 < `50ms`（硬件依赖）。
- UART 吞吐：`115200` 下连续发送 1 分钟，误码率 ≈ `0`，无缓冲溢出。

**日志与度量采集**
- 启用 `MC_StartDebugLog(0)` 输出连接与错误日志；记录时间戳与返回码。
- 抓包过滤 `udp.port == <端口>`；统计丢包与重传（如有应用级计数）。
- Web UI 前端控制台：记录 `test.progress/test.completed` 到达间隔与总时长。

**故障排查流程**
- 步骤 1：电源与指示灯检查 → 线缆与 RJ45 接触 → 急停线路（常闭低有效）。
- 步骤 2：IP 与网段验证 → 端口一致性（PC/卡一致，尝试 `0` 或 `60000`）。
- 步骤 3：防火墙放行 `UDP` → 关闭冲突网卡 → `ping` 与 `arp -a` 基本连通性。
- 步骤 4：`MC_Open` 返回码分析：`-6/-7/-8` 对应连通/超时/通信打开失败；查看调试日志。
- 步骤 5：接口/符号问题：改用 `.lib` 静态链接；以 C++ 成员函数为准；避免错误 `extern "C"`。
- 步骤 6：适配层与对象生命周期：`MultiCard` 栈上创建；传递时空删除器；避免崩溃。
- 步骤 7：Wireshark 抓包确认是否有报文往返；必要时更换端口或网卡。

**通过/失败判定**
- 通过：所有基础功能用例均通过；性能指标满足；无严重错误码或崩溃日志。
- 失败：出现连续超时、错误码 `-6/-7/-8` 不能排除、IO/串口不工作或安全异常；需提交缺陷与日志。
