# 开发规范

## 分层原则

- `domain` 不直接依赖具体硬件、文件系统或 UI。
- `application` 负责编排流程，不承载底层实现细节。
- `infrastructure` 负责端口实现，不反向泄漏到领域层。
- `adapters` 只负责输入输出转换，不写核心业务规则。

## 编码原则

- 优先小目标、小接口、小提交。
- 新公共类型进入 `shared` 前必须确认其稳定性。
- 新错误码、事件名、状态字段必须同步文档化。

## 变更原则

- 跨层引用前先判断是否应改为 port/adapter。
- 不允许为赶进度绕过契约层直接访问其他子项目内部实现。
- 涉及真实硬件行为变更时必须补最小验证路径。
