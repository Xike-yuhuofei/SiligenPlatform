# 新窗口接续摘要

使用这个模板生成 handoff 文档。内容要短、具体、可执行，不要写成长篇分析。

## 文件位置

- 目录：`docs/session-handoffs/`
- 文件名：`YYYY-MM-DD-HHMM-<task-slug>-session-handoff.md`

## 写作规则

- 只写当前会话里已经明确出现的信息。
- 推断必须单独标记，不得混进事实段落。
- 如果某项没有可靠信息，直接写“未知”或“未提供”。
- 每个章节优先使用 2 到 6 条短句或短条目。

## 固定结构

```md
# 新窗口接续摘要

## 1. 任务本质

## 2. 当前阶段

## 3. 当前目标

## 4. 已确认事实

## 5. 已完成事项

## 6. 未完成事项

## 7. 建议新窗口立刻执行的下一步

## 8. 关键文件 / 文档 / 路径 / 命令

## 9. 关键约束 / 风险 / 阻塞

## 10. 推断 / 未知
```

## 最小建议内容

- `任务本质`：1 到 2 句，说清当前在解决什么。
- `当前阶段`：例如“已完成分析，准备实现”“已完成实现，等待回归”“已收敛根因，待最小修复”。
- `当前目标`：本轮新窗口最直接要达成的目标。
- `已确认事实`：只写已从当前会话明确确认的事实。
- `已完成事项`：已经执行并收口的动作、验证、决策。
- `未完成事项`：尚未实施、尚未验证、尚未确认的部分。
- `建议新窗口立刻执行的下一步`：最好是 1 到 3 个具体动作。
- `关键文件 / 文档 / 路径 / 命令`：只列最关键的，不做全量清单。
- `关键约束 / 风险 / 阻塞`：明确边界条件和当前卡点。
- `推断 / 未知`：把推断和未知集中列出，避免污染事实层。
