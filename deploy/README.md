# deploy

`deploy/` 是部署与交付资产的正式承载根（canonical root）。

## 正式职责

- 承载部署材料、环境投放约束与交付清单。
- 承载发布流程所需的交付说明与可审计约束定义。
- 作为历史分散部署资产的目标收敛面，统一部署事实口径。

## 非职责边界

- 不承载业务实现源码与业务 owner 逻辑。
- 不替代 `scripts/` 的自动化脚本职责。
- 不承载测试报告、临时运行日志或一次性排障产物。

## 迁移与回流约束

- 部署说明、发布约束与交付清单应从历史位置逐步收敛到 `deploy/`。
- 在迁移未完成前，可保留历史入口作为兼容说明，但应显式指向 `deploy/`。
- 新增部署交付类资产默认写入 `deploy/`，禁止写回历史分散路径。
