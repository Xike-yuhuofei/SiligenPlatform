# shared/kernel

`shared/kernel/` 是 `shared/` 下承载低业务含义基础能力与跨模块内核契约的 canonical 子根。

## 正式职责

- 承载跨模块复用且不归属单一业务模块的基础能力（ID、envelope、失败语义、消息骨架、日志抽象等）的公共内核定义。
- 提供可被 `modules/*`、`apps/*`、`tests/*` 复用的稳定基础类型与约束，不直接表达业务流程。
- 作为工作区唯一正式共享基础内核根，统一承载公共实现与公共约束。

## 不应承载

- 一级业务 owner 逻辑、业务编排与流程状态机实现。
- 仅服务单一模块且无跨模块复用价值的私有实现。
- 未声明生命周期的临时兼容代码或隐式 fallback。

## 迁移与退出规则

- 新增公共内核实现必须优先落在 `shared/kernel/`。
- 不允许在 canonical 根之外长期维持第二套共享基础事实源。
- 兼容 wrapper 若仍存在，必须可审计并具备明确退出门禁。
