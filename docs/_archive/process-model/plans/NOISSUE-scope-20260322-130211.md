# NOISSUE Scope - 整仓目录重构前置准备（适配版候选骨架）

- 状态：Frozen
- 日期：2026-03-22
- 分支：feat/dispense/NOISSUE-e2e-flow-spec-alignment
- 输入来源：用户确认采用候选目录骨架方向，并要求先落实重构前准备工作
- 阶段：ss-scope-clarify

## 1. 背景与目标

当前仓库已经形成以 `apps/`、`packages/`、`integration/`、`tools/`、`config/`、`data/`、`examples/` 为核心的 canonical 工作区骨架，并围绕这些路径固化了 build/test/CI、配置、数据和运行入口。

本任务不是立即执行物理目录迁移，而是先把“按候选目录骨架重构整仓”所需的前置准备冻结成正式工件，避免后续目录重构在 owner、门禁、资产路径和回滚条件上边做边想。

本阶段目标：

1. 冻结“适配版候选骨架”的目标边界，而不是字面照搬候选目录。
2. 冻结当前 canonical 目录到目标骨架的映射关系。
3. 冻结重构前必须完成的阻塞项、基线门禁与回滚单元。
4. 输出可直接指导后续架构与执行阶段的范围、架构方案和测试计划。

## 2. In Scope

1. 明确本次目录重构的目标形态：
   - 保留根级 `config/`、`data/`、`examples/`、`integration/`、`tools/`、`docs/`
   - 业务代码骨架向候选文档的 `apps/ + modules/ + shared/` 方向收敛
2. 盘点当前目录、owner、硬编码路径、build/test/CI 门禁与 residual legacy 依赖。
3. 给出当前 `apps/*`、`packages/*` 到目标骨架的映射与分阶段迁移顺序。
4. 冻结准备阶段的验收标准、风险、前置依赖和不做项。
5. 输出对应的架构方案与测试计划文档。

## 3. Out of Scope

1. 本阶段不做任何物理目录迁移或代码 owner 切换。
2. 不修改 `build.ps1`、`test.ps1`、`ci.ps1`、`.github/workflows/*` 的实现。
3. 不修改运行时代码对 `config/`、`data/`、`examples/` 的默认解析路径。
4. 不在本阶段重新设计 DXF、运行时、设备适配、仿真链路的业务逻辑。
5. 不对候选文档目录本身做归档、入库或格式整理。

## 5. 验收标准（可测试、可观察）

1. 本任务必须新增一组与本次目录重构专门对应的规划工件：
   - `NOISSUE-scope-20260322-130211.md`
   - `NOISSUE-arch-plan-20260322-130211.md`
   - `NOISSUE-test-plan-20260322-130211.md`
2. 目标骨架必须明确说明：
   - 采用“适配版候选骨架”
   - 保留根级 `config/`、`data/`、`examples/`、`integration/`、`tools/`
   - 不做一次性字面目录翻转
3. 架构方案必须覆盖当前 `apps/*`、`packages/*` 的目标归宿与迁移波次，不能留“后续再看”的灰区。
4. 架构方案必须显式记录当前 `control-core`、`third_party`、`src/shared`、`src/infrastructure`、`modules/device-hal` 的阻塞事实。
5. 测试计划必须给出根级命令、报告目录和预期结果，能够作为重构前基线冻结的执行输入。
6. 文档必须显式声明本阶段不做物理迁移、不改默认运行资产路径、不改对外入口命令。

## 6. 风险与前置依赖

1. `control-core` 仍承担顶层 CMake/source-root owner、`third_party`、`src/shared`、`src/infrastructure`、`modules/device-hal` 等真实职责；若不先盘清这些依赖，目录重构会制造新的双 owner 状态。
2. 当前 CMake、PowerShell、Python、GitHub Actions、架构文档和运行脚本对现有 canonical 路径有大量硬绑定，目录重构不能仅靠 `Move` 文件解决。
3. `config/`、`data/`、`examples/` 已是默认运行资产路径；若把它们回卷进候选 `shared/` 或 `samples/` 语义，会直接破坏运行与发布口径。
4. 当前工作树存在未跟踪的候选规格文档目录；在其未进入版本控制前，它只能作为参考输入，不能作为受门禁保护的正式基线。

## 7. 明确不做项

1. 不在本阶段移动 `apps/*`、`packages/*`、`integration/*`、`config/*`、`data/*` 的 repo-tracked 文件。
2. 不在本阶段新增新的默认运行入口、默认配置入口或默认数据入口。
3. 不在本阶段删除任何 legacy 目录或 compatibility shell。
4. 不在本阶段把 `packages/*` 简单批量重命名为 `modules/*`。

## 8. 默认决策（已冻结）

1. 目标骨架：采用适配版候选结构，不做候选文档的字面照搬。
2. 切换节奏：采用分阶段切换，不做一次性整仓 cutover。
3. 运行资产：保留根级 `config/`、`data/`、`examples/`、`integration/`、`tools/`、`docs/` 为 canonical。
4. 对外入口：保留当前 `apps/control-runtime`、`apps/control-tcp-server`、`apps/control-cli`、`apps/hmi-app` 的外部命名与入口契约。
