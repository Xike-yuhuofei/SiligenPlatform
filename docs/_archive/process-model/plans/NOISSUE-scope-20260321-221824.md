# NOISSUE Scope - process-runtime-core 并发与状态一致性修复（独立审查闭环）

- 状态：Frozen
- 日期：2026-03-21
- 分支：`feat/hmi/NOISSUE-dispense-trajectory-preview-v2`
- 输入来源：`packages/process-runtime-core` 独立审查报告（P0/P1/P2/P3）
- 阶段：`ss-arch-plan` 基线

## 1. 目标

在不扩展业务功能的前提下，修复 `packages/process-runtime-core` 中已识别的高风险缺陷，重点消除：

1. 线程生命周期悬空与潜在 UAF。
2. 状态机假成功/假取消/假超时。
3. 取消、暂停、清理的时序不一致。
4. 关键安全检查空实现导致的误放行风险。
5. 可测试性缺口导致的回归不可观测问题。

## 2. In Scope

1. `src/application/usecases/dispensing/*`
2. `src/application/usecases/motion/monitoring/*`
3. `src/application/usecases/redundancy/*`
4. `src/infrastructure/adapters/redundancy/*`
5. `src/domain/dispensing/domain-services/ValveCoordinationService*`
6. 上述改动对应的 `tests/unit/*`（新增/补全并发与状态机测试）

## 3. Out of Scope

1. 新业务能力开发（新工艺、新协议、新 UI 功能）。
2. `process-runtime-core` 以外的大规模架构重构。
3. 全链路发布流程改造与生产运维平台建设。
4. 非审查项触达的性能专项优化（仅保留风险兜底）。

## 4. 模块边界约束

1. `application/usecases` 仅编排端口与状态机，不引入 runtime-host 具体实现。
2. `domain` 继续保持设备能力抽象，不回写宿主线程管理逻辑。
3. `infrastructure/adapters` 仅承接存储与调度实现，不向上层泄露实现细节。
4. 禁止新增跨包反向依赖，遵循 `packages/process-runtime-core/BOUNDARIES.md`。

## 5. 成功标准（验收口径）

1. 不出现可复现的析构后后台线程访问已释放对象问题。
2. `Cancel/Pause/Resume/Stop/Cleanup` 在竞态下返回语义与真实状态一致。
3. 关键失败路径包含可检索上下文（至少 failure_stage / error_code / message）。
4. 空安全检查路径被阻断或被完整实现，不允许默认放行。
5. 单测覆盖新增竞态与终态一致性场景；回归命令可在仓库根目录复现。

## 6. 风险与依赖

1. 调度器与 usecase 的生命周期契约当前不完整，修复时需同步定义“任务停止已确认”语义。
2. 并发修复可能改变原有时序，必须通过新增测试防回归。
3. 冗余仓储写入策略调整涉及数据完整性，需提供可回滚路径。
