## 1. 使用说明

本矩阵只回答 4 件事：

1. 当前目录在目标骨架中的归宿是什么。
2. 该目录在目录重构中的定位是“保留 / 拆分 / 并入 / 过渡保留”哪一种。
3. 该目录属于哪个迁移波次。
4. 进入该波次前必须先消除哪些阻塞项。

本矩阵是后续目录迁移任务拆分的主索引。未在此矩阵登记的目录，不允许私自进入物理迁移。

## 8. 进入物理迁移前的目录准备门槛

1. 本矩阵中的每一行都必须有唯一目标归宿，禁止空白。
2. 任何一行若仍存在“长期保留在 `packages/`”的表述，则说明该 owner 尚未冻结，不能进入物理迁移。
3. 对 `config/`、`data/`、`examples/`、`integration/`、`tools/` 的任何翻转提议，一律视为超出本轮适配版候选骨架范围。
4. `control-core` residual dependency 未迁出前，`packages/process-runtime-core`、`packages/device-adapters`、`packages/device-contracts` 只允许做 owner 冻结和路径盘点，不允许做最终目录切换。
