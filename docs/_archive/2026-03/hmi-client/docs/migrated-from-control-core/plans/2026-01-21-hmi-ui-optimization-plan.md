# HMI UI 优化实施计划

## 背景

基于 ChatGPT 的界面分析建议，结合代码审查结果，制定以下优化计划。

## 代码审查发现

| 建议 | 代码现状 | 结论 |
|------|----------|------|
| IO状态用RadioButton | 已使用LED样式 (`role="led-off"`) | **无需修改** |
| 状态显示 `[]` | 使用 `[*]` / `[ ]` 文本 | **需要修改** |
| 阀门缺乏状态反馈 | 已有状态标签和role切换 | **无需修改** |

## 实施任务

### P0 - 高优先级

#### 1. 轴状态显示改为LED样式
- **文件**: `src/adapters/hmi/ui/main_window.py`
- **位置**: 第 637-645 行（创建）、第 1524-1529 行（更新）
- **改动**:
  - 将 `QLabel("[ ]")` 改为 LED 样式的 `QLabel`
  - 更新逻辑从 `setText("[*]")` 改为 `_update_led(led, "green"/"off")`

### P1 - 中优先级

#### 2. 进度条对比度优化
- **文件**: `src/adapters/hmi/ui/styles.py`
- **位置**: 第 382-385 行
- **改动**:
  - 将 `#2196F3` (蓝色) 改为 `#4CAF50` (绿色) 或更亮的蓝色 `#42A5F5`

#### 3. 急停按钮样式统一
- **文件**: `src/adapters/hmi/ui/main_window.py`
- **位置**: 侧边栏急停按钮
- **改动**:
  - 统一为红色实心大按钮样式，与顶部急停一致

### P2 - 低优先级

#### 4. 断开连接增加确认弹窗
- **文件**: `src/adapters/hmi/ui/main_window.py`
- **位置**: `_on_disconnect` 方法（第 1120-1131 行）
- **改动**:
  - 添加 `QMessageBox.question()` 确认对话框

#### 5. DXF范围格式化
- **文件**: `src/adapters/hmi/ui/main_window.py`
- **位置**: DXF 文件信息显示区域
- **改动**:
  - 将 `X: 0-0, Y: 0-0` 改为 `X: 0.0 ~ 150.0 mm` 格式

## 不实施的建议

| 建议 | 原因 |
|------|------|
| 移除侧边栏急停 | 工业安全设计需要冗余急停 |
| 暂停按钮降低饱和度 | 高饱和度是工业警示色标准 |
| 目标数量位置上移 | 当前位置可防止误操作，需用户调研 |
| 隐藏执行参数空列 | 该功能可能后续实现 |

## 实施顺序

```
P0-1 (轴状态LED) → P1-2 (进度条) → P1-3 (急停样式) → P2-4 (断开确认) → P2-5 (DXF格式)
```
