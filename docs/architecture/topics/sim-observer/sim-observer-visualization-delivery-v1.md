# 模拟器可视化界面开发与验收说明 V1

> 本说明以《数据契约规格说明 V1》为唯一数据契约来源。

## 1. 目标与定位

最终目标是开发 `simulation-engine` 对应的可视化界面。

该界面的产品定位不是“给仿真内核补一个附属 UI”，而是建设一层基于既有确定性 recording/export 事实的**可视化观察层**，服务：

- 轨迹检查
- 几何结果确认
- 工艺路径审查
- 人工调试
- 问题复盘
- 演示培训

## 2. 首批范围

### 2.1 `P0` 主目标

首批先完成“看结果”的闭环，优先解决：

- 轨迹合理性确认
- 几何结果确认
- 点位与段顺序检查
- 覆盖关系审查
- 结果异常定位

### 2.2 `P1` 预留与补齐

结构上必须支持“看过程”，首批后续补齐：

- 执行过程复盘
- 报警前后状态分析
- 人工调试支持
- 培训与讲解

### 2.3 首批明确不做

- 物理工艺仿真
- 自动根因分析
- 自动修复建议
- 自动工艺质量裁决

## 3. 首批工作面

### 3.1 默认工作面

`P0` 默认工作面固定为：

- 结果摘要视图
- 结果画布视图
- 路径结构视图
- 上下文详情视图

默认判断顺序：

1. 先看问题入口
2. 再看空间事实
3. 再看序列事实
4. 必要时再进入过程复盘

### 3.2 按需进入工作面

`P1` 工作面包括：

- 时间过程视图
- 报警上下文视图

它们不抢占 `P0` 默认首屏。

## 4. 视图职责

### 4.1 结果摘要视图

负责：

- 问题分流
- 优先级排序
- 结果入口

不负责：

- 单独证明问题真实性
- 自动根因解释

### 4.2 结果画布视图

负责：

- 空间结果判断
- 异常位置确认
- 覆盖关系观察

不负责：

- 顺序精确认定
- 时间播放

### 4.3 路径结构视图

负责：

- 顺序关系
- 段间衔接
- 对象定位

不负责：

- 全量点位常驻展示
- 整体空间直观判断

### 4.4 上下文详情视图

负责：

- 单对象、单问题、单时间锚点的事实展开

不负责：

- 主导航
- 全局总览

### 4.5 时间过程视图

负责：

- 关键过程节点推进
- 时间到对象或区间回接

不负责：

- 终态总览
- 自动根因判断

### 4.6 报警上下文视图

负责：

- 报警前提与直接后果观察
- 报警对象或对象集合入口

不负责：

- 统一异常中心
- 整场历史重放

## 5. 主操作链路

首批行为级主链路固定为：

- `S1` 从摘要确认结果异常
- `S2` 从画布确认路径是否异常
- `S3` 从结构确认顺序与衔接
- `S4` 从报警项复盘前后窗口
- `S5` 从时间锚点理解结果如何形成

链路级底线：

- `P0` 必须在摘要、画布、结构之间形成短闭环
- `P1` 必须能回接对象、对象集合或顺序区间
- 任一链路都必须支持成功终止、失败降级与回退
- 任一链路的终点都必须能被用户一句话转述为可行动判断

## 6. 开发说明

### 6.1 `D2` `P0` 默认工作面

交付目标：

- 结果摘要视图 V1
- 结果画布视图 V1
- 路径结构视图 V1
- 上下文详情视图 V1
- `P0` 组合工作面

完成标准：

- `S1-S3` 可运行
- 四视图消费同一对象身份与状态语义
- `P0` 闭环不依赖 `P1`

### 6.2 `D3` 联动、回退与空态

交付目标：

- 强联动规则
- 弱联动规则
- 派生联动规则
- 回退路径规则
- 空态与降级规则
- `P0` 导航闭环

完成标准：

- `S1-S3` 默认路径都可运行
- 所有失败路径都可解释、可回退、不可伪造

### 6.3 `D5` 失败保护与不可接受状态拦截

交付目标：

- 缺锚点处理规则
- 映射失败处理规则
- 集合未收敛处理规则
- 错误终点拦截规则
- 不可接受状态拦截规则

完成标准：

- `P0` 的失败路径被系统化、类型化、可回退地接住
- 红线状态被显式拦截，不得静默继续

### 6.4 `D4` `P1` 工作面

交付目标：

- 时间过程视图 V1
- 报警上下文视图 V1
- `P1 -> P0` 回接机制
- `P1` 失败保护
- `P1` 默认进入与退出规则

完成标准：

- `S4-S5` 可运行
- 时间与报警分析能回接对象或明确暴露不足
- `P1` 不抢占 `P0` 默认首屏

### 6.5 `D6` 统一行为级验收

交付目标：

- `P0` 验收场景集
- `P1` 验收场景集
- 失败分支验收集
- 不可接受状态拦截验收集
- 验收记录模板
- 验收结论矩阵

完成标准：

- `P0`：`S1-S3` 全通过，且无全局不可接受状态
- `P1`：`S4-S5` 至少完成锚点、回接、失败保护契约验证

## 7. 推荐执行顺序

推荐按以下顺序推进：

1. `D2 + D3 + D5` 先完成 `P0`
2. `D4` 完成 `P1`
3. `D6` 统一验收 `S1-S5`

这也是当前首批开发与联调的正式执行顺序。

## 8. 开发执行计划

### 8.1 迭代划分

- `I0` 基线落地
  - 把 `D1` 固化到代码中的统一类型、状态、规则结果壳和载荷入口
- `I1` `P0` 工作面骨架
  - 让四个 `P0` 视图可见、可消费同一批数据、可共享选中态
- `I2` `P0` 联动闭环
  - 完成联动、回退、空态和失败保护，跑通 `S1-S3`
- `I3` `P1` 工作面
  - 完成时间过程与报警上下文，跑通 `S4-S5`
- `I4` 统一验收与收尾
  - 执行 `D6`，形成发布结论

### 8.2 里程碑

- `M0` 契约冻结可消费
- `M1` `P0` 可见但未闭环
- `M2` `P0` 闭环可验收
- `M3` `P1` 可用
- `M4` 首批发布候选
