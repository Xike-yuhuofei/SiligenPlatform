# DXF Closed Contour Direction Semantics V1

- 状态：Proposed
- 日期：2026-04-06
- 适用范围：`apps/hmi-app`、`apps/runtime-gateway`、`modules/job-ingest`、`modules/topology-feature`、`modules/process-path`、`modules/motion-planning`、`modules/runtime-execution`

## 1. 目的

冻结闭合 DXF 轮廓在 SiligenSuite 中的方向语义，消除以下歧义：

- DXF 原始实体方向是否默认构成执行约束
- 闭合轮廓是否允许在执行前重写段方向、起点和连接顺序
- HMI 预览与在线执行在闭合轮廓语义上是否必须一致

本文不直接给出代码补丁，只定义产品语义、研发约束和排查入口。

## 2. 问题定义

在在线机台测试中，`Demo-1.dxf` 的闭合外轮廓出现明显折返；同一场景下，HMI 动态路径预览未观察到同类现象。

这说明当前至少存在两个需要冻结的问题：

1. 闭合轮廓在产品上应被视为“几何输入”还是“方向受限输入”
2. 预览链路与执行链路是否消费了不同层级或不同顺序的路径数据

当前需求已明确收敛为：

- 对所有闭合轮廓，系统默认允许方向重排
- 只要最终几何边界保持等价，段方向、连接顺序和起点都可以被算法重写
- 优化目标是减少空走、减少折返、提升路径连续性与走胶连续性

## 3. 语义冻结

### 3.1 DXF 是几何输入，不是默认方向约束输入

DXF 在本系统中提供的是几何意图。对闭合轮廓而言：

- 原始绘制顺序不是默认执行真值
- 原始段方向不是默认执行真值
- 原始起点不是默认执行真值

若后续存在必须固定方向的特殊工艺，必须通过显式工艺参数表达，不能隐式继承 DXF 原始方向。

### 3.2 闭合轮廓默认允许重排

对所有闭合轮廓，系统默认允许：

- 翻转单段方向
- 重建闭合链遍历顺序
- 重新选择轮廓起点

默认目标不是“尽量还原 DXF 录入方式”，而是“尽量形成连续加工路径”。

### 3.3 不变量

闭合轮廓在任意重排后都必须保持以下不变量：

- 几何边界等价
- 闭合性不被破坏
- 不漏段
- 不断链
- 不错连
- 不引入额外几何偏移

## 4. 优先级与非目标

### 4.1 优化目标优先级

1. 减少空走
2. 减少折返
3. 提升路径连续性
4. 提升走胶连续性

### 4.2 非目标

本文不覆盖：

- 开放轮廓的方向策略
- 阀控时序、材料特性、供胶波动
- 机械精度、标定误差、控制卡插补精度验收
- 具体速度规划和角点减速参数

## 5. 主链语义边界

### 5.1 输入几何层

职责：

- 读取 DXF 线段、弧段及其拓扑关系

约束：

- 可保留原始方向作为源数据参考
- 不得把原始方向提升为执行硬约束

### 5.2 几何归一化层

职责：

- 将原始实体整理成稳定的闭合轮廓表示

约束：

- 必须允许方向翻转、闭合链重建、起点重选
- 这一层之后，不应再把“原始导入顺序”当成默认工艺真值

### 5.3 工艺路径层

职责：

- 为闭合轮廓生成面向加工的连续路径

约束：

- 必须以连续性优先，而不是以 DXF 原始方向保真优先
- 若闭合轮廓仍直接沿用导入顺序执行，应视为与本文语义冲突

### 5.4 运动规划层

职责：

- 基于既定工艺路径生成速度与轨迹规划结果

约束：

- 不得重新从 DXF 原始实体恢复方向语义
- 只应承接上游已冻结的路径顺序

### 5.5 执行段生成层

职责：

- 将规划结果切分为控制器可执行段

约束：

- 不得凭空引入新的高层折返语义
- 段切分可以改变执行粒度，但不应重写闭合轮廓的上游遍历意图

### 5.6 HMI 展示层

职责：

- 展示几何预览、工艺路径预览或执行路径预览

约束：

- 必须明确展示绑定的是哪一层数据
- 若预览展示的是执行路径语义，则必须与真实执行顺序一致
- 若预览仅展示几何或理想工艺路径，则必须允许其与执行追踪明确区分

## 6. 研发实现约束

### 6.1 必须满足

1. 闭合轮廓在归一化或工艺路径阶段必须进入“可重排”语义
2. 执行链路不得默认继承 DXF 原始方向
3. 闭合轮廓的最终执行顺序必须由连续性目标驱动
4. 预览链路和执行链路必须能明确对齐或明确区分
5. 若存在方向强约束场景，必须通过显式工艺参数注入

### 6.2 明确禁止

- 以 DXF 原始段方向作为闭合轮廓默认执行约束
- 以 DXF 原始录入顺序作为闭合轮廓默认执行顺序
- 在执行下发前无文档约束地再次重建另一套路径顺序
- 用 HMI 理想预览掩盖在线执行链路的真实段序偏差

## 7. 验收标准

满足以下条件时，可认为本文语义已被实现：

1. 对任意闭合轮廓，系统默认不以 DXF 原始方向为执行约束
2. 闭合轮廓允许重写方向、起点和连接顺序，且几何边界保持等价
3. 在线执行不再出现无工艺必要的频繁折返
4. 预览与执行若展示同一语义层，则段序一致
5. 预览与执行若展示不同语义层，则系统文档和代码责任均能清楚说明

## 8. 排查入口与证据要求

建议优先按以下顺序排查：

1. HMI 动态路径预览的数据源
2. 在线执行下发的数据源
3. 闭合轮廓归一化逻辑
4. 工艺路径排序逻辑
5. 执行段生成与网关发送逻辑

每个闭合轮廓至少应采集以下证据字段：

- 轮廓 ID
- 段 ID
- 段类型
- 原始起点、终点
- 原始方向
- 归一化后方向
- 工艺路径最终顺序
- HMI 预览使用顺序
- 在线下发使用顺序
- 最终执行段发送顺序

若以上字段无法对齐，不能宣称已定位首次偏离层。
