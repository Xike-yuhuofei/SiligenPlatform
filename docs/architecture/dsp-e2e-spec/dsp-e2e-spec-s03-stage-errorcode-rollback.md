# 《点胶机端到端流程规格说明 s03：阶段失败码与回退策略表（修订版）》

本版把失败码、阻断、首件分流、回退目标、人工放行边界统一到一套可实现规则中，用于直接指导：

- 状态机阻断/终止迁移
- UI 报警分层
- 自动恢复策略
- 追溯与统计
- 测试矩阵设计
- 人工审批边界
- 回退与 `ArtifactSuperseded` 执行策略

本版与修订后的阶段、对象、接口、状态机口径保持一致，特别修正：

1. S11 拆分为 S11A 执行包组装、S11B 离线校验；
2. S12 失败定义为**执行门禁阻断**，默认不回退规划层；
3. S13 失败必须按“空跑错 / 出胶错 / 位置偏 / 审批缺失”分流；
4. 所有回退目标都使用正式枚举，而不是自然语言；
5. 失败事件载荷与命令总线公共字段对齐。

# 1. 失败码与失败载荷设计规则

## 1.1 失败码命名规则

统一采用：

```text
<阶段>_<类别>_<序号>
```

示例：
- `S2_PARSE_001`
- `S5_RULE_004`
- `S11B_PKG_002`
- `S12_READY_005`
- `S15_FAULT_003`

## 1.2 类别建议

| 类别 | 含义 |
|---|---|
| `CTX` | 任务上下文错误 |
| `FILE` | 文件接收/文件结构错误 |
| `PARSE` | 解析失败 |
| `GEOM` | 几何对象异常 |
| `TOPO` | 拓扑异常 |
| `FEAT` | 特征提取异常 |
| `RULE` | 工艺规则/模板异常 |
| `COORD` | 坐标/基准/变换异常 |
| `ALIGN` | 对位/补偿异常 |
| `PATH` | 工艺路径异常 |
| `TRAJ` | 运动轨迹异常 |
| `TIME` | 点胶时序异常 |
| `PKG` | 执行包/离线校验异常 |
| `READY` | 设备未就绪 |
| `QUAL` | 首件/质量门失败 |
| `FAULT` | 运行时故障 |
| `TRACE` | 归档/追溯失败 |

## 1.3 失败事件统一载荷

所有失败/阻断事件建议至少带以下字段：

| 字段 | 说明 |
|---|---|
| `job_id` | 归属任务 |
| `workflow_id` | 工作流主键 |
| `stage_id` | 失败发生阶段 |
| `artifact_ref` | 关联对象引用 |
| `failure_code` | 机器可读失败码 |
| `severity` | 严重等级 |
| `retryable` | 是否可重试 |
| `manual_override_allowed` | 是否允许人工放行 |
| `rollback_target` | 推荐回退目标 |
| `requires_new_job_or_version_chain` | 是否必须新建任务或新版本链 |
| `blocking_scope` | 阻断范围 |
| `requires_manual_decision` | 是否必须人工决策 |
| `execution_context` | `execution_package_ref / execution_id / machine_id / segment_index` |
| `recommended_action` | 推荐动作 |
| `message` | 面向人的说明 |
| `occurred_at` | 发生时间 |

## 1.4 严重等级建议

为与总线载荷保持一致，建议 `severity` 固定为：

| 等级 | 含义 | 典型处理 |
|---|---|---|
| `S0` | 安全/数据完整性/硬阻断问题，不允许放行 | 立即阻断或中止 |
| `S1` | 关键正确性问题，不允许带病进入下一层 | 阻断并回退/修复 |
| `S2` | 可恢复阻断，修复条件后可重试 | 停留当前层或恢复 |
| `S3` | 评估类告警，不影响正确性主链 | 记录、提示、必要时人工评估 |

说明：
- `severity` 用于运行时与事件系统，不等价于文档评审问题级别，但可复用命名。
- 只有 `S3` 类评估告警才可能进入人工放行讨论。

## 1.5 回退目标枚举（与状态机统一）

统一使用：
- `ToSourceAccepted`
- `ToGeometryReady`
- `ToTopologyReady`
- `ToFeaturesReady`
- `ToProcessPlanned`
- `ToCoordinatesResolved`
- `ToAligned`
- `ToProcessPathReady`
- `ToMotionPlanned`
- `ToTimingPlanned`
- `ToPackageBuilt`
- `ToPackageValidated`

# 2. 失败处理总规则

## 2.1 可重试，不等于可继续生产
例如：
- `S7_ALIGN_001` 对位识别失败：通常可重试；
- `S12_READY_003` `baseline invalid`：不能靠“重试一次”过关；
- `S13_QUAL_002` 首件堆胶：不能靠重跑一遍掩盖。

## 2.2 人工放行只覆盖评估类告警
原则上禁止人工放行：
- 文件结构非法
- 拓扑不稳定
- 工艺模板缺失
- 坐标基准错误
- 对位补偿超限
- 轨迹越界/干涉
- 时序非法
- 设备安全链未满足
- 首件质量未通过
- 运行时严重故障

仅建议有限放行的场景：
- 非安全、非正确性、非质量门的估算类告警
- 统计/展示类异常，但不影响执行正确性

## 2.3 上游产物一旦修改，必须新建任务或完整新版本链
若改变了以下任一对象：
- `FeatureGraph`
- `ProcessPlan`
- `CoordinateTransformSet`
- `AlignmentCompensation`
- `ProcessPath`
- `MotionPlan`
- `DispenseTimingPlan`
- `ExecutionPackage`

则原则上不得沿用旧下游产物，必须显式产生新版本链，并对旧链发出 `ArtifactSuperseded`。

## 2.4 S12 设备未 ready，不触发规划回退
`PreflightBlocked` 只说明：
- 现在不能执行；
- 不是规划错；
- 不是包失效；
- 不应重算路径/轨迹/工艺。

## 2.5 S13 首件拒绝后，必须先分流再回退
禁止直接：
- `FirstArticleRejected -> StartExecutionCommand`
- `FirstArticleRejected -> 强行人工放行（无审批对象）`
- `FirstArticleRejected -> 随机回退一个层级`

S11A 的问题优先留在组包层解决，不能直接跳去设备层试跑。

# 4. 失败处理级别建议（用于状态机与 UI）

| 级别 | 含义 | 典型阶段 | 对应状态特征 |
|---|---|---|---|
| `L1-输入阻断` | 不允许进入后续阶段 | S0-S5 | 停在当前阶段 |
| `L2-规划阻断` | 上游数据不成立，必须重建下游产物 | S6-S11B | 触发回退与 `ArtifactSuperseded` |
| `L3-执行门禁阻断` | 规划可用，但设备暂不可执行 | S12 | `PreflightBlocked` |
| `L4-首件质量阻断` | 路径/工艺/补偿存在实物风险 | S13 | `RollbackPending` |
| `L5-运行时故障` | 执行中断，需恢复或终止 | S14-S15 | `Recovering / Aborted / Faulted` |

# 5. 人工放行硬边界

## 原则上禁止人工放行
- 文件结构非法
- 解析失败
- 拓扑歧义
- 特征未分类
- 工艺模板缺失
- 坐标基准错误
- 对位补偿超限
- 轨迹越界/干涉
- 开阀/关阀时序非法
- 设备安全链未满足
- 首件质量未通过
- 运行时严重故障

## 可有限人工放行
仅建议考虑：
- S11B 的非安全、非质量关键估算类告警
- 统计/展示类异常，但不影响执行主链

结论：
**人工放行只覆盖“评估类告警”，不覆盖“正确性类失败”。**

# 6. 最关键的工程结论

## 6.1 失败码不能只服务“报错显示”
它还必须服务：
- 回退策略
- 自动恢复
- 测试设计
- 故障统计
- 质量复盘

## 6.2 “可重试”和“可放行”必须分开
这两个概念混在一起，系统一定会失控。

## 6.3 S12、S13、S15 必须严格区分
- S12：门禁问题
- S13：质量门问题
- S15：运行时故障问题

这三类问题的责任层、回退目标和治理方式完全不同。

## 6.4 真正高价值的是“失败归属层 + 正式回退目标”
不是报错文案，而是系统能否稳定回答：
- 这是文件问题？
- 几何/拓扑问题？
- 工艺问题？
- 坐标/对位问题？
- 路径/轨迹问题？
- 时序问题？
- 设备门禁问题？
- 运行时故障问题？
