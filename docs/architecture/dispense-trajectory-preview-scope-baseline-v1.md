# 点胶轨迹预览范围冻结基线 V1

- 状态：Frozen
- 日期：2026-03-21
- 分支：`feat/hmi/NOISSUE-dispense-trajectory-preview-v2`
- 适用阶段：P0（首轮交付）

## 1. 任务本质

在正式点胶执行前，提供可被操作者直接判定的轨迹预览能力，用于提前发现路径、顺序和边界问题，降低误执行风险。  
本轮不以工艺效果优化为目标，而以“执行前可判断”作为唯一业务导向。

## 2. 核心目标

- 将点胶轨迹从“内部数据”转为“可视化可确认对象”。
- 在执行前形成明确的人工确认闸口。
- 建立单一输入来源与一致性验收口径，避免预览与实际执行语义漂移。

## 3. 成立条件

- 预览结果可支持操作者做“是否允许执行”的判断。
- 预览与执行在几何路径与顺序语义上保持一致。
- 同一输入下预览结果可复现、可比对、可追溯。
- 操作者可明确判断当前预览来源，避免将 mock 占位结果误判为真实 runtime 轨迹。

## 4. 非目标

- 不包含轨迹生成算法重构或优化。
- 不包含胶量、铺展等工艺结果一致性承诺。
- 不包含真实设备联动作为本轮必选交付项。
- 不包含多工件、复杂异常恢复场景的全面覆盖。

## 5. 约束与边界（冻结）

### 5.1 六项歧义冻结结论

1. 准确性口径：以“几何一致 + 顺序一致”为准，不纳入工艺结果一致。
2. 触发时机：支持编辑后手动预览；执行前必须完成一次预览确认。
3. 数据来源：统一采用“下发前最终轨迹快照”。
4. 用户动作：仅“查看 + 确认签核”，不含预览内直接编辑。
5. 场景范围：仅覆盖单工件、单任务、标准完整流程（不含异常中断恢复）。
6. 链路范围：本轮仅覆盖仿真/可视化链路，不将真实设备链路作为必选验收。

### 5.2 架构与契约边界

- 架构边界调整：不做跨域重构；以现有模块边界内增量实现为前提。
- 状态机/生命周期：引入“执行前预览确认”业务关口，视为流程门禁语义变更。
- 外部契约/接口/文件格式：本轮原则上不新增外部文件格式；若发现现有契约无法承载，需单独立项评审后变更。
- 测试基线：新增并固化“几何一致、顺序一致、确认门禁”三类验收基线。

## 6. 输入与输出定义

- 输入：
  - 下发前最终轨迹快照（唯一数据源）
  - 当前任务上下文（工件、坐标语义、任务标识）
  - 预览触发动作（手动触发/执行前强制）
- 输出：
  - 可视化轨迹预览结果（用于人工判定）
  - 预览确认状态（是否完成执行前确认）
  - 预览关联记录（用于追溯）

## 7. 影响范围

- 必然影响：`hmi` 交互与展示域。
- 条件影响：任务流程状态语义（执行前门禁）。
- 暂不纳入：真实设备执行链路、工艺闭环验证链路。

## 8. 已知风险

- 预览语义与执行输入若出现偏差，会直接削弱门禁有效性。
- 若流程门禁定义不清，可能引入“可执行状态”判定歧义。
- 若追溯记录不足，现场问题难以复盘。

## 9. 依赖项

- 下发前最终轨迹快照可稳定获取。
- 任务上下文标识可用于预览结果关联追溯。
- 现有流程具备承载“执行前确认”语义的位置。

## 10. 验收口径

- 需求验收：
  - 预览能力可用于执行前人工判断。
  - 预览确认可作为执行前门禁条件。
- 一致性验收：
  - 几何一致、顺序一致口径成立。
  - 数据来源唯一且可追溯。
  - 预览响应必须能区分 `planned_glue_snapshot`、旧版 `runtime_snapshot` 与 `mock_synthetic`。
- 真实轨迹验收：
  - 涉及真实轨迹外观、几何或顺序签收时，只接受 `planned_glue_snapshot + glue_points` 数据源。
  - `mock_synthetic` 仅可用于联调和 smoke，不得作为真实轨迹签收依据。
- 范围验收：
  - 严格限定在冻结范围内，无新增未评审目标。
- 基线验收：
  - 本文档作为 P0 范围基线，后续变更需显式记录版本与原因。

## 11. P1 契约冻结（2026-03-21）

- 新增命令：`dxf.preview.snapshot`
  - 请求：与 `dxf.plan.prepare` 共享轨迹参数来源，至少提供 `dispensing_speed_mm_s`（兼容别名 `speed_mm_s`）
  - 响应：`snapshot_id`、`snapshot_hash`、`segment_count`、`point_count`、`total_length_mm`、`estimated_time_s`、`generated_at`
  - 错误码：`3010`、`3011`、`3012`、`3013`、`3014`
- canonical 执行链：`dxf.artifact.create -> dxf.plan.prepare -> dxf.preview.snapshot -> dxf.preview.confirm -> dxf.job.start`
  - `dxf.preview.confirm` 负责显式锁定用户确认的快照
  - `dxf.job.start` 作为唯一正式启动入口，不再通过 `dxf.execute` 兼容链承载执行语义

## 12. P2 优先级调整（2026-03-21）

- 暂停仿真链路的轨迹图渲染作为执行前门禁依赖。
- HMI 预览优先走真实链路 `dxf.preview.snapshot` 生成的 `planned_glue_snapshot + glue_points`。
- 当前阶段在预览区展示真实链路胶点主预览与辅助执行折线摘要（段数/点数/长度/时长/快照哈希），确认签核与执行前哈希复验保持不变。

## 13. P3 数据源标识补充（2026-03-26）

- `dxf.preview.snapshot` 响应新增 `preview_source` 字段：
  - `planned_glue_snapshot`：来自 planning/workflow 共享 authority 的正式胶点快照，可进入真实轨迹验收口径。
  - `runtime_snapshot`：旧版轨迹快照，仅保留给诊断/兼容链，不得进入成功主预览。
  - `mock_synthetic`：来自 mock 占位数据，仅可用于联调，不代表真实 DXF 几何或真实点胶轨迹。
- HMI 必须显式显示 `preview_source`，并在 `mock_synthetic` 时给出强提示。
- 验收流程必须拒绝仅凭 `mock_synthetic` 结果签收“真实轨迹预览通过”。
