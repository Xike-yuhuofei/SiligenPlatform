# 首层场景筛选稿

## 文档定位

本稿基于已冻结的《工程替身系统总纲说明页》《需求定义稿》《非目标边界》《成立条件稿》《范围定义稿》和《核心范围分层稿》，只回答一个问题：

**用哪些少量高判别力场景，来证明首层最小研发闭环真实成立。**

本稿不回答实现方案、模块划分、接口设计、技术路线和排期。

## 一、筛选目标

首层场景筛选的目标，不是先追求覆盖广度，而是用少量高判别力场景，证明以下事实已经成立：

- 研发可以在不依赖真机的情况下进入关键验证上下文
- 研发可以对关键流程、关键分支、关键异常和关键数据变化做出有效判断
- 关键集成问题能够在上机前暴露
- 同一验证场景能够稳定重演，并在相同条件下保持结果一致性
- 验证结果能够比较，并形成明确结论，从而进入日常回归与自动化入口

## 二、筛选原则

### 1. 直接服务首层目标

入选场景必须直接服务“脱离真机推进、前移暴露问题、形成稳定验证闭环”三项目标中的至少一项，最好同时覆盖多项。

### 2. 尽量跨越多条成立条件

入选场景必须能跨越多条成立条件，而不是只证明一个局部点。

### 3. 缺失后会迫使团队回真机

若某场景缺失会迫使团队回到真机完成关键判断，则该场景优先进入首层证明集。

### 4. 优先高频、高风险、高耦合

入选场景优先选择高频、高风险、高分歧、高集成耦合的场景，而不是低频长尾场景。

### 5. 必须形成可判定闭环

不能形成可重复、可比较、可判定闭环的场景，不应进入首层证明集。

## 三、成立条件映射简写

为便于后续表格引用，本稿使用以下简写：

- `C1`：主动构造验证上下文
- `C2`：提供有效反馈
- `C3`：验证关键交互时序、控制链路与操作序列
- `C4`：主动构造异常并检验恢复路径
- `C5`：在数据化场景组合下确认行为
- `C6`：稳定重演、结果可比较且结论可判定

## 四、首层最小证明集

在当前阶段，首层证明集冻结为以下 `9` 个高判别力业务代表场景。

| 场景 | 场景名称 | 代表语义 | 覆盖成立条件 | 纳入理由 |
| --- | --- | --- | --- | --- |
| S1 | 主干成功闭环 | 已连接、已回零、已激活配方、DXF 有效，完成“路径规划 → 点胶执行 → 结果输出”主干闭环 | C1 C2 C3 C5 C6 | 证明系统能承接一条完整研发验证链，而不是只支持局部观察或局部注入 |
| S2 | 前置条件阻断与有效反馈 | 未激活配方或未选择有效版本时，尝试启动 DXF 点胶任务被阻断，并明确说明阻断原因 | C1 C2 C5 C6 | 证明系统能判断“为什么不能开始”，而不是只会跑通 happy path |
| S3 | 前置条件阻断与有效反馈 | 未回零、供胶未开或关键设备状态未就绪时，启动任务被拒绝，并给出具体卡点 | C1 C2 C3 C6 | 证明设备就绪判断与执行前置条件可在脱机环境下成立 |
| S4 | 前置条件阻断与有效反馈 | DXF 无效、空轨迹或关键轨迹参数无效时，规划或执行前被阻断并给出原因 | C1 C2 C5 C6 | 把“设计输入 → 路径生成”这段前移到首层，而不是只盯执行阶段 |
| S5 | 关键操作链路切换 | 主干任务执行中发生“暂停 → 继续 → 停止/复位 → 再进入”，状态流转与再进入行为正确 | C1 C2 C3 C6 | 证明首层不是单一路径验证环境，而是能承接关键操作链路 |
| S6 | 关键交互时序异常 | 至少一个跨模块或跨接口边界上出现超时、错序或关键响应缺失，任务进入明确可判定状态 | C1 C2 C3 C6 | 这是“前移暴露集成问题”的首层证明场景，缺它就退化成本地逻辑调试 |
| S7 | 高频或高风险异常与恢复 | 点胶阀、供胶链路或关键传感信号异常触发报警，经过处理后恢复到正确状态或安全停止 | C1 C2 C4 C6 | 证明异常处理与恢复路径已被承接，而不是只验证正常流程 |
| S8 | 高频或高风险异常与恢复 | 运行中急停后进入锁定状态，随后完成复位、重新入场或明确终止判定 | C1 C2 C3 C4 C6 | 证明高风险异常路径不再只能依赖真机联调 |
| S9 | 关键数据组合对照 | 同一 DXF/任务在两组关键配方版本、关键参数或关键配置下执行，对结果差异进行可比、可判定对照 | C1 C2 C5 C6 | 证明系统具备真实的数据化场景承接能力，而不是只能跑单一默认配置 |

## 五、筛选结论

当前首层不应先追求“场景多”，而应先用这 `9` 个高判别力业务代表场景，证明最小研发闭环真实成立。

后续若继续推进，应基于本稿继续把每个场景压成：

- 逐项说明
- 场景-成立条件映射
- 验收口径
- 评审记录模板

但不应在未冻结本稿前，直接进入更细的逐项证明材料。

## 六、本稿结论

当前可冻结的结论是：

首层场景筛选应围绕“首层必需承接”展开，而不是围绕广覆盖展开。
在当前阶段，`S1-S9` 这 `9` 个高判别力业务代表场景，构成首层最小证明集，并可作为后续逐项说明、验收口径和正式评审的上位输入。
