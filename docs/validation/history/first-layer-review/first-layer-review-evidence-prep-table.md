# S1-S9 首轮评审证据准备表

## 使用说明

本表用于首层正式评审前的证据准备检查。  
任一场景若缺少“判定基线”或“证据来源”，默认不得进入正式判定。  
任一场景若无法说明“为什么它补的是首层缺口”，默认不得进入正式判定。
本表中的“初步结论候选”与“是否可开评”判断，应以《验收口径稿》为准。

## 字段说明

- 业务前提：该场景成立所依赖的业务前置条件
- 验证上下文：进入评审时应处于的上下文与状态
- 关键输入条件：本轮评审使用的输入与触发条件
- 判定基线：本次判断“算通过 / 不算通过”的基准
- 证据来源：日志、状态快照、对照输出、结果记录、结论引用等
- 真实跨边界交互：是否涉及真实跨模块、跨接口或对端交互边界
- 重复验证：是否已完成重复验证，可支撑稳定重演判断
- 初步结论候选：通过 / 不通过 / 有条件通过 / 待补证据
- 是否可开评：是 / 否
- 核心覆盖条件：应能回映到《场景-成立条件映射稿》中该场景对应的核心覆盖条件

## 开评前快速检查

- 是否每个场景都明确了首层证明责任
- 是否每个场景都写明了对应首层缺口
- 是否每个场景都具备可引用的证据来源
- 是否每个场景都具备明确判定基线
- 是否不依赖临场口头解释才能完成结论判断

## 结论规则

- `是否可开评 = 否`：
  - 判定基线缺失
  - 证据来源缺失
  - 场景证明责任不清
  - 无法说明它补的是哪类首层缺口
- `是否可开评 = 是`：
  - 输入、基线、证据与证明责任均已齐备
  - 可进入正式首层评审
