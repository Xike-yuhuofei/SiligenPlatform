# 阻断结论总表

## 文档定位

本表用于统一管理首层正式评审中识别出的阻断项。
它回答的不是“问题多不多”，而是：

- 哪些缺口已经构成对“首层成立”的直接阻断
- 每个阻断项对应哪个场景或哪类首层缺口
- 阻断解除前，哪些结论不能被判为成立

## 阻断定义

以下情况通常应判为阻断项：

- 关键场景未完成其首层证明责任
- 关键判断仍需回真机才能完成
- 关键结论无法由证据直接支撑，仍依赖临场解释
- 结论不可稳定重演，无法支撑可比较、可判定闭环
- 某类首层缺口没有对应承担场景或承担场景未成立

## 缺口类型建议口径

- `证据缺失`：没有足够证据支撑判定
- `基线不清`：算通过 / 不算通过的判定基线不明确
- `上下文不足`：未能构造到有效验证上下文
- `交互未成立`：真实跨边界交互或关键链路未成立
- `异常恢复未成立`：异常构造或恢复路径未被证明
- `可重演性不足`：结果不可稳定重演或不可比
- `范围覆盖缺失`：某类首层缺口无人承担或承担不成立

## 使用规则

- 任一“是否阻断首层成立 = 是”且“当前状态 = 未解除”的条目存在时，整体结论不得判为 `首层成立`
- 单场景可以判 `有条件通过`，但若其边界缺口已命中阻断标准，仍应在本表登记阻断项
- 阻断解除必须对应明确的证据补齐或重审结论，不能仅以口头说明关闭
