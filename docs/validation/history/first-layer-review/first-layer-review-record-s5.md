## 一、场景说明

- 业务前提：主干任务已正常进入执行态，且具备触发暂停/继续/停止/复位的条件
- 验证上下文：应处于真实任务运行过程中，而不是静态模拟状态切换
- 关键输入条件：应明确操作顺序、触发时点和再进入条件
- 证明责任摘要：证明关键操作链路在运行中可被正确承接，且再进入行为可判定
- 成功边界：状态切换顺序正确、再进入行为正确、无非法跨态跳转
- 失败边界：状态错乱、复位后残留旧态、或再进入逻辑不可判定

## 二、范围裁剪

- 是否仍属首层必需：以《核心范围分层稿》为准；当前默认应为“是”
- 是否可被其他场景覆盖：通常不应被主干成功闭环替代，因为它关注运行中操作链路
- 是否存在弱场景替代风险：重点防止只验证单个按钮动作而不验证完整操作链
- 裁剪结论：仍属首层必需，本轮不得用单个动作响应替代完整操作链。
