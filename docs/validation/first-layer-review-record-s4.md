## 一、场景说明

- 业务前提：应刻意构造 DXF 无效、空轨迹或关键轨迹参数异常的输入
- 验证上下文：系统进入规划前或执行前的数据校验阶段
- 关键输入条件：应明确具体无效输入类型及其触发方式
- 证明责任摘要：证明设计输入质量问题能在首层被识别并阻断，而不是等到真机执行时才暴露
- 成功边界：无效输入被正确拒绝，原因明确且可复现
- 失败边界：无效输入漏过校验、原因模糊、或无法定位到输入问题

## 二、范围裁剪

- 是否仍属首层必需：以《核心范围分层稿》为准；当前默认应为“是”
- 是否可被其他场景覆盖：通常不应被 `S2` 或 `S3` 覆盖，因为它关注设计输入质量
- 是否存在弱场景替代风险：重点防止只做离线文件检查而不验证规划/执行前真实阻断
- 裁剪结论：仍属首层必需，本轮不得用离线文件检查替代真实规划前阻断。

## 三、验收判定

- 判定基线：输入问题在正确阶段被阻断，且阻断原因足以支撑研发修正
- 证据来源：建议包含输入样本、校验结果、阻断反馈与相关日志
- 是否涉及真实跨边界交互：按实际填写
- 是否完成重复验证：建议重点核对
- 六项必判项：
  - 证明责任是否被真实触发：
  - 核心成立条件是否被覆盖：
  - 结果是否足以支撑研发判断：
  - 是否补上首层缺口：
  - 是否可比较、可判定：
  - 是否具备稳定重演基础：
- 是否补上首层缺口：应对应“设计输入与轨迹前置校验缺口”
