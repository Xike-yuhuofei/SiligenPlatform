## 一、场景说明

- 业务前提：任务已在运行中，且急停链路、锁定状态与复位路径可被触发
- 验证上下文：应在真实运行态触发急停，而不是静态假设急停后状态
- 关键输入条件：应明确急停触发时点、复位动作、重新入场或终止策略
- 证明责任摘要：证明高风险异常下的锁定、复位和重新入场/终止判定链路可被承接
- 成功边界：急停后进入正确锁定态，后续复位/重新入场/终止判定清楚且可重演
- 失败边界：急停后状态不受控、复位后残留非法状态、或终止判定含糊
