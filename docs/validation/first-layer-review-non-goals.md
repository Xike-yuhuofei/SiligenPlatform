# 非目标边界

## 文档定位

本稿用于把“工程替身系统当前明确不承接什么”单独冻结下来。
它的作用不是否定这些内容永远没有价值，而是明确：这些内容不构成当前系统在定义上成立所必须承接的边界。

本稿只做边界排除，不讨论实现方式、优先级或未来扩展路线。

## 一、判断原则

如果某项内容的主要价值不在于承接研发流程中的关键判断责任，不在于帮助关键判断尽可能脱离真机完成，不在于形成可重复、可比较、可判定的研发闭环，那么它就不应进入当前阶段的核心目标。

换句话说，以下内容即使未来可能有价值，只要它不直接服务当前定义基线，就不应被写成系统成立所必须承接的部分。

## 二、明确非目标

### 1. 展示型目标不是当前目标

当前不以“看起来更像设备”作为目标。
因此，不把以下内容写入成立条件：

- 外观展示完整度
- 运行画面表现力
- 演示友好的可视化效果
- 面向汇报或对外展示的交互包装

### 2. 三维、空间形态与数字孪生导向不是当前目标

当前不以构建三维仿真、空间建模或数字孪生系统为方向。
因此，不把以下内容写入成立条件：

- 三维场景还原
- 空间布局与几何形态完整建模
- 设备世界的数字映射完整性
- 为“更真实”而做的建模扩展

### 3. 全量设备世界复制不是当前目标

当前不以“尽可能完整复制真实设备世界”作为默认方向。
因此，不要求：

- 把所有设备部件、状态、过程和边界一次性纳入
- 先追求覆盖广度再判断是否形成研发闭环
- 以“真实系统有什么，就全部仿什么”作为需求来源

### 4. 最终物理验收替代不是当前目标

当前系统不是为了替代真机完成最终物理验收、工艺验收或最终性能确认。
因此，不把以下内容纳入成立条件：

- 最终工艺效果的真实性担保
- 最终物理指标的验收结论
- 最终设备性能的替代确认
- 对真实现场最终结论的完全替代

### 5. 培训、演示、对外呈现不是当前目标

当前系统不是培训系统，也不是演示系统。
因此，不以以下内容作为首层成立依据：

- 培训便利性
- 演示易用性
- 面向非研发角色的可理解性优化
- 对外展示价值

### 6. 既有在研仿真系统不是需求来源

当前不能以已有在研仿真系统的能力、模块、边界或实现习惯反推真实需求。
因此，以下做法应被明确排除：

- 以现有系统已有能力直接当作“必须需求”
- 以现有系统缺失能力直接当作“可以不做”
- 以现有实现边界直接当作“合理边界”
- 以现有架构、接口或技术方案反向塑形定义基线

## 三、边界使用方式

本稿冻结后，后续文稿应按以下方式使用：

- 需求定义稿用于说明系统为什么存在、必须承接什么、如何判断成立
- 本稿用于说明哪些方向明确不应进入当前成立条件
- 成立条件稿用于把“必须承接什么”压成门槛式判定条件

若后续文稿出现“展示型”“数字孪生型”“全量复制型”或“既有系统反推型”表述，应优先以本稿为准进行裁剪，而不是为了实现便利性保留。

## 四、本稿结论

当前阶段要冻结的，不是一个尽可能完整、尽可能逼真的设备仿真目标，而是一套面向研发流程的工程替身系统定义基线。

因此，展示效果、三维导向、全量设备复制、最终物理验收替代、培训演示价值，以及既有在研仿真系统的能力边界，都不属于当前系统在定义上成立所必须承接的内容。
