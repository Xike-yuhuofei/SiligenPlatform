# 证据补齐与首轮复审准备稿

## 文档定位

本稿用于把首轮正式评审后识别出的“待补证据”状态，收束成一份可执行的补证据与复审准备文稿。

本稿不重开定义边界，不新增场景，不重写验收口径，只回答四个问题：

- 当前到底缺什么证据
- 缺口分别对应哪个场景与哪个阻断项
- 补证据时至少要补到什么程度
- 何时可以进入首轮复审

## 一、当前状态结论

当前首轮正式评审的实际状态是：

- `S1-S9` 全部已进入正式记录，但当前结论均为 `待补证据`
- 整体结论当前只能判为 `待补证据后复审`
- [阻断结论总表](/D:/Projects/SiligenSuite/docs/validation/first-layer-review-blocker-matrix.md) 中的 `B1-B3` 均为未解除

因此，当前阶段的工作重点不再是扩写评审文稿，而是补齐能直接支撑判定的正式运行证据。

## 二、补证据总原则

### 1. 只补能直接支撑判定的证据

- 不补与当前证明责任无关的外围材料
- 不补只增加“看起来更完整”但不能改变判定的低价值材料

### 2. 每条证据都必须可回溯到场景与阻断项

- 补齐的证据必须能明确回答“它是为哪个场景补的、为哪个阻断项解除服务”
- 不允许出现大量无法映回 `S1-S9` 或 `B1-B3` 的泛化材料

### 3. 先补正式运行证据，再谈重审

- 若仍没有正式运行结果、状态记录、异常记录或对照结果，就不应进入复审
- 复审不是再开一次讨论会，而是在证据齐备条件下重做判定

### 4. 重复验证证据必须纳入同一轮准备

- 本轮阻断项里 `B3` 直接要求可比较、可判定、可重演闭环成立
- 因此补证据时不能只补一次性结果，必须同步考虑重复验证材料

## 三、阻断项拆解与补证据目标

### B1 主干闭环、操作链路与数据对照的正式运行证据不足

- 对应场景：`S1`、`S5`、`S9`
- 当前缺口：
  - `S1` 尚缺主干闭环运行结果、结果输出和重复验证记录
  - `S5` 尚缺操作链路切换、状态流转、再进入结果和重复验证记录
  - `S9` 尚缺两组关键数据组合的对照结果和重复验证记录
- 最低补齐目标：
  - 至少拿到一次正式运行结果
  - 至少拿到一次可直接引用的状态/输出记录
  - 至少拿到一次重复验证记录或成组对照记录
- 解除判断：
  - 若三类场景均不再停留于“仅有代码入口与文稿描述”，才可考虑解除 `B1`

### B2 交互异常、工艺异常与急停链缺少正式异常处置证据

- 对应场景：`S6`、`S7`、`S8`
- 当前缺口：
  - `S6` 尚缺超时、错序或缺响应异常的交互记录与状态结果
  - `S7` 尚缺异常触发、报警、处理动作与恢复/安全停止记录
  - `S8` 尚缺急停后的锁定、复位、重新入场或终止记录
- 最低补齐目标：
  - 至少拿到一次异常触发记录
  - 至少拿到一次异常后的状态结果记录
  - 至少拿到一次异常处理或恢复链闭合记录
- 解除判断：
  - 若三类异常链均已有直接可判定证据，才可考虑解除 `B2`

### B3 全部场景尚未形成可比较、可判定、可重演的正式结论闭环

- 对应场景：`S1-S9`
- 当前缺口：
  - 当前所有场景都缺少足以支撑“稳定重演”和“结果可比较”的正式证据
- 最低补齐目标：
  - 每类关键场景至少补一组重复验证结果
  - 数据对照类场景至少补一组可比输出
  - 异常类场景至少补一组异常触发后的稳定结果
- 解除判断：
  - 只有当整体不再依赖一次性现象做判断时，才可考虑解除 `B3`

## 五、复审输入包要求

首轮复审前，建议每个待复审场景至少补齐以下输入：

- 本次复审要解除的阻断项或待补证据项
- 复审所用输入条件
- 复审新增证据清单
- 复审前后的差异说明
- 复审判定候选结论

若某场景无法明确说明“这次新增证据到底解除哪一条缺口”，则不应进入复审。

## 六、何时允许进入首轮复审

### 场景级进入条件

单个场景允许进入复审，至少应满足：

- 判定基线已明确
- 证据来源已齐备
- 当前待补缺口已对应到具体新增证据
- 复审目标是重新判断场景结论，而不是继续澄清定义

### 整体级进入条件

整体允许进入首轮复审，至少应满足：

- `B1-B3` 都已有明确补证据动作与待核验输入
- `S1-S9` 不再全部处于“只有待补说明、没有新证据”的状态
- 复审会议输入能够直接支撑单场景重新判定

## 七、复审执行顺序建议

建议按阻断收敛效率排序，而不是按原始场景编号机械排序：

1. `S1`、`S5`、`S9`
   先收敛 `B1`
2. `S6`、`S7`、`S8`
   再收敛 `B2`
3. `S2`、`S3`、`S4`
   最后补齐三类前置阻断场景
4. 整体复核 `B3`
   最后判断是否已形成可比较、可判定、可重演闭环

## 八、本稿结论

当前可冻结的结论是：

- 当前首层正式评审的下一步，不是再讨论定义，而是围绕 `B1-B3` 做证据补齐
- 首轮复审准备的核心不是“再开一次会”，而是先把每个场景的正式运行、异常处置、对照输出和重复验证材料补到可判定程度
- 后续若继续推进，应以本稿为执行入口，逐项消减 `待补证据` 与未解除阻断项
