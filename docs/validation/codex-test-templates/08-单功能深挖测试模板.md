# 单功能深挖测试模板

```text
请在当前仓库中，针对“[这里填功能名]”做一次“无机台深挖测试”，并直接执行，不要只给方案。

目标：
验证该功能在无机台条件下的软件链路、协议调用、门禁拦截和领域规则是否正常。

要求：
1. 不使用真实机台，不连接真实硬件。
2. 优先复用仓库内已有的 mock、smoke、unit test、ctest、协议测试入口。
3. 不要停在分析，直接执行测试。
4. 如现有覆盖不足，可补最小必要测试并执行。
5. 如测试跑不起来，先定位并修复测试入口或构建产物，再继续。
6. 全程使用简体中文。

请至少覆盖以下内容：
1. 正常路径
2. 前置条件检查
3. 互锁拦截
4. 停止/关闭/释放等收尾动作
5. HMI 层、协议层、领域层是否一致

请按以下顺序推进：
1. 查找相关实现与测试入口
2. 跑最接近真实链路的 smoke 或 HMI 测试
3. 跑协议层 + mock 联调
4. 跑领域层或聚合单测中的对应用例
5. 如果缺测试，补最小必要测试并执行

最终输出：
1. 使用了哪些命令和入口
2. 实际执行结果
3. 明确结论：通过 / 失败 / 阻塞
4. 当前无机台测试覆盖了什么
5. 当前无机台测试没有覆盖什么
```
