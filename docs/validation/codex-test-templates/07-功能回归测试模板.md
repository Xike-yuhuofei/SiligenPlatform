# 功能回归测试模板

```text
请在当前仓库中，对以下功能做一次“无机台回归测试”，并直接执行到拿出明确结论为止：

功能列表：
1. 点动
2. 回零
3. 点胶阀控制
4. 供胶控制
5. 急停
6. move/move_to
7. 安全门、急停、限位对上述功能的互锁拦截

要求：
1. 不使用真实机台，不连接真实硬件。
2. 优先复用仓库内已有的 mock server、smoke 脚本、单元测试、集成测试、ctest、协议测试入口。
3. 不要只分析，直接执行。
4. 不要因为第一次失败就停止，要继续定位、修复路径、补构建产物、切换正确入口后再执行。
5. 不要默认全量构建；如需构建，只构建最小必要目标。
6. 先不要改业务代码；只有现有测试覆盖明显不足时，才允许补最小必要测试。
7. 全程使用简体中文。

执行方式：
1. 先扫描仓库，定位每项功能相关的 HMI、协议、mock、领域层测试入口。
2. 按以下顺序优先执行：
   - HMI/UI smoke
   - 协议层 + mock 联调
   - 领域层/规则层单元测试
3. 每项功能至少覆盖：
   - 正常路径
   - 前置条件检查
   - 互锁拦截
   - stop/release/close 收尾动作
4. 如果测试在聚合测试二进制里，必须定位正确目标并实际执行。
5. 如果缺少测试产物，先补构建再执行。

最终输出：
1. 已通过
2. 失败或异常
3. 未覆盖项
4. 汇总表
```
