# 文件树 + 业务解读文档工具

## 1. 用途

这个工具用于整理任意业务目录，输出一份文档，回答两件事：

- 目录结构是什么
- 实现文件在指定业务流程中做什么

## 2. 输入项

每次执行前，先明确 3 个输入：

| 输入项 | 说明 | 示例 |
|---|---|---|
| `目标目录` | 本次要整理的目录 | `modules/` |
| `业务主线` | 所有说明围绕的业务流程 | `点胶流程` |
| `结果文件` | 最终输出文档路径，由当前任务显式指定 | `docs/process-model/reviews/<task>-file-tree-business.md` |

- `结果文件` 不再允许复用仓库级固定总表。
- `modules/MODULES_BUSINESS_FILE_TREE_AND_TABLES.md` 已退役，不能再作为默认输出或声明性边界依据。

## 3. 输出形式

结果文档固定分两部分：

### 3.1 文件树

- 只展示目录和文件结构
- 不写文件说明

### 3.2 表格

- 按最后一级文件夹分组
- 每个叶子目录一张表
- 只解释实现文件在业务流程中的作用

## 4. 纳入规则

### 4.1 文件树

纳入：

- `.cpp`、`.cc`、`.cxx`
- `.h`、`.hpp`
- `.py`
- `.json`、`.yaml`、`.yml`、`.toml`
- `CMakeLists.txt`
- 元数据文件，例如 `module.yaml`

排除：

- `.md`
- `.gitkeep`
- `.pyc`
- `.txt`
- `.def`
- `.exp`

### 4.2 表格

默认只纳入实现文件：

- `.cpp`
- `.cc`
- `.cxx`
- `.py`

默认排除：

- `CMakeLists.txt`
- 元数据文件
- 所有 `.h/.hpp`
- 文档、缓存、占位文件

## 5. 表格字段

表格固定使用 5 列：

| 列名 | 要求 |
|---|---|
| `流程阶段` | 写业务阶段，不写“领域层/适配层” |
| `触发条件` | 写清楚什么时候进入这段逻辑 |
| `当前职责` | 写清楚这个文件直接做什么 |
| `输出结果` | 写清楚结果交给谁或给下一段做什么 |
| `失败影响` | 写清楚出错后的现场表现或链路后果 |

## 6. 写作规则

每条说明都必须围绕业务主线，只回答这 5 个问题：

- 位于哪一段
- 什么时候触发
- 当前做什么
- 输出什么
- 失败会怎样

以下写法不合格：

- “这是某某类的实现文件”
- “负责某某逻辑”
- “用于处理相关业务”
- “位于某某目录，承担某某能力”

如果没有流程位置、触发条件、输出结果或失败影响，就要重写。

## 7. 执行步骤

### 步骤 1：确认输入

- 确认目标目录
- 确认业务主线
- 确认结果文件路径

### 步骤 2：生成文件树

- 扫描目录
- 按文件树规则过滤
- 输出纯结构树

### 步骤 3：识别叶子目录

- 找出所有包含实现文件的最后一级目录
- 每个叶子目录对应一张表

### 步骤 4：逐文件填写表格

对每个实现文件，补齐：

- `流程阶段`
- `触发条件`
- `当前职责`
- `输出结果`
- `失败影响`

### 步骤 5：复查

逐行检查：

- 有没有业务阶段
- 有没有触发条件
- 有没有具体动作
- 有没有下游输出
- 有没有失败后果

## 8. 模板

### 8.1 结果文档头部模板

```md
# <目标目录名称> 文件树与业务解读

本文档分两部分：

- 文件树：展示 `<目标目录>` 的代码结构，不附文件说明。
- 表格：按最后一级文件夹展示实现文件在 `<业务主线>` 中的具体作用。

- 目标目录：`<目标目录>`
- 业务主线：`<业务主线>`
- 表格列：`流程阶段`、`触发条件`、`当前职责`、`输出结果`、`失败影响`
- 表格默认纳入：`.cpp`、`.cc`、`.cxx`、`.py`
- 表格默认排除：`CMakeLists.txt`、元数据文件、`.h/.hpp`、文档、缓存、占位文件
```

### 8.2 文件树模板

````md
## <分组或模块名>

**文件树**
```text
<目标目录或模块名>/
├── <子目录>/
│   ├── <文件A>
│   └── <文件B>
└── <文件C>
```
````

### 8.3 表格模板

```md
**<叶子目录路径>/**

| 文件 | 流程阶段 | 触发条件 | 当前职责 | 输出结果 | 失败影响 |
|---|---|---|---|---|---|
| `<文件名>` | `<位于哪一段>` | `<什么时候进入>` | `<直接做什么>` | `<输出给谁/下一段做什么>` | `<出错后会怎样>` |
```

### 8.4 草稿模板

```text
流程阶段：位于 <业务流程阶段>。
触发条件：当 <触发条件> 时进入。
当前职责：负责 <当前动作>。
输出结果：输出 <输出对象>，交给 <下游对象/下游环节>。
失败影响：如果这里出错，会导致 <现场表现/链路后果>。
```

## 9. 当前实例

当前仓库不再维护固定总表实例文件。

- 每次执行都必须为当前任务单独指定结果文件路径。
- 推荐示例：`docs/process-model/reviews/<task>-file-tree-business.md`
