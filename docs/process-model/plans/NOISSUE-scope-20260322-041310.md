# NOISSUE Scope - HMI 点胶轨迹预览界面去冗余与画布扩展

- 状态：Frozen
- 日期：2026-03-22
- 分支：`feat/hmi/NOISSUE-dispense-trajectory-preview-v2`
- 输入来源：用户需求（HMI 点胶轨迹预览 UI 精简）
- 阶段：`ss-scope-clarify`

## 1. 背景与目标

当前 HMI 点胶轨迹预览区域包含标题“真实链路轨迹预览（点状胶点）”及解释性文案，并在画布下方展示较多参数/指标，导致可视区域被压缩。目标是：

1. 删除预览标题与对应说明文案。
2. 删除与实际生产决策无关的参数/指标展示。
3. 调整信息区布局为可多列排布（在仍需展示必要信息时），尽量扩大画布可视面积。
4. 明确保证“执行前确认”和“快照哈希校验”行为不变。

## 2. In Scope

1. HMI 点胶轨迹预览 HTML 片段结构调整（标题/说明删除、信息区瘦身、布局改为多列能力）。
2. 预览画布尺寸策略调整（提升画布高度占比与可用区域优先级）。
3. 与预览展示耦合的单元测试同步更新（仅针对 UI 输出与关键门禁不回归）。
4. 预览确认与哈希校验链路的回归验证（行为不变性验证）。

## 3. Out of Scope

1. 轨迹点采样算法、点云渲染算法、坐标映射逻辑重写。
2. `dxf.preview.confirm` / `dxf.execute` 协议字段与网关校验规则变更。
3. 生产流程控制逻辑（启动、暂停、急停、互锁）改造。
4. 非点胶轨迹预览页面的通用 UI 重构。

## 4. 受影响模块清单（apps/packages/integration/tools）

1. apps
   - `apps/hmi-app/src/hmi_client/ui/main_window.py`（`_build_preview_html`）
   - `apps/hmi-app/tests/unit/*preview*.py`（预览 UI 与门禁相关测试）
2. packages
   - 预期无代码改动；仅做“确认无行为漂移”验证：`packages/transport-gateway`（哈希校验链路）
3. integration
   - 预期无新增集成代码；如有需补充 UI 冒烟或回归证据到 `integration/reports/*`
4. tools
   - 预期无工具脚本改动

## 5. 验收标准（可测试、可观察）

1. 预览 HTML 不再包含以下文案：
   - “真实链路轨迹预览（点状胶点）”
   - “轨迹图按快照点集以点状方式渲染，模拟真实胶点分布；执行前确认与哈希校验仍生效。”
2. 画布区域可视占比提升：同等窗口下，轨迹 SVG 显示高度较基线实现增加（以 UI 快照或尺寸断言证明）。
3. 画布下方信息区不再展示“与实际生产无关”的指标；保留字段应满足“启动/确认判定直接需要”原则。
4. 信息区布局支持多列（在存在多项信息时不强制单列表格堆叠）。
5. 启动前门禁仍严格生效：
   - 未确认预览时启动被阻断。
   - 预览快照哈希不一致时启动被阻断。
   - 相关阻断提示语义不弱化。
6. 现有与 `snapshot_hash`、`preview_confirm`、`execute` 相关单元测试持续通过，或在等价语义下更新后通过。

## 6. 风险与前置依赖

1. “非生产相关指标”定义若不明确，可能引发反复修改；需以“是否直接影响启动/确认判定”为裁剪标准。
2. 过度压缩信息区可能影响现场排障；需保留最小可追踪字段（建议：`snapshot_id` / `snapshot_hash` / 确认态）。
3. UI 文案与 DOM 结构调整会影响快照类测试与字符串断言测试，需同步修订测试基线。
4. 画布放大后在低分辨率窗口可能导致滚动行为变化，需要最小窗口尺寸下可用性回归。

## 7. 明确不做项

1. 不修改后端协议、网关哈希算法或执行链路鉴权。
2. 不引入新的预览交互能力（缩放、拖拽、图层开关等）。
3. 不改造点胶工艺参数计算或执行速度模型。
4. 不在本阶段处理跨页面视觉统一。

## 8. 关键澄清问题（待确认，最多 5 项）

1. “与实际生产无关”的判定是否采用以下默认口径：仅保留直接影响执行门禁与追溯的字段（如 `snapshot_id`、`snapshot_hash`、确认态），其余指标默认移除？
2. 多列排布是否接受响应式规则：宽屏双列、窄屏单列？
3. 画布扩展的目标优先级是否为“固定优先画布高度”，即信息区在必要时折行/换列而非占用画布高度？
