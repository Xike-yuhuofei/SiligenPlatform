# NOISSUE Scope - 轨迹预览点状化与交点小尾巴修复

- 状态：Frozen
- 日期：2026-03-22
- 分支：feat/hmi/NOISSUE-dispense-trajectory-preview-v2
- 输入来源：用户缺陷反馈（预览仍呈线段感、交点存在小尾巴）
- 阶段：ss-scope-clarify

> 冻结基线：NOISSUE-scope-20260322-042851.md

## 1. 背景与目标

现状问题有两类：
1. 轨迹预览预期为点状几何，但视觉效果仍接近连续线段。
2. 线段交点附近出现“小尾巴”伪影，影响几何可信度。

本阶段目标：
1. 锁定“点状显示”的工程定义（不是主观观感）。
2. 锁定“小尾巴”的可检测判据与修复边界。
3. 形成可直接进入实现与回归的验收标准。

## 2. In Scope

1. 预览点集生成与抽样链路核查与修正（从 planning/trajectory 到 preview snapshot）。
2. 交点“小尾巴”根因定位与修复（重点关注回折点、角点过冲、抽样别名问题）。
3. HMI 预览渲染策略修正，确保点状呈现可观察且可测试（不退化成视觉连续线）。
4. 相关单元测试与回归测试补齐（至少覆盖 rect_diag.dxf）。
5. 真实 runtime 链路验证（非 mock 数据）。

## 3. Out of Scope

1. 点胶执行控制逻辑改造（启停、急停、互锁、阀门时序）。
2. 新增复杂交互能力（缩放、拖拽、图层开关、3D 视图）。
3. 全量 DXF 几何引擎重写或运动学算法全面替换。
4. 非轨迹预览页面的 UI 统一改版。

## 5. 验收标准（可测试、可观察）

1. 点状渲染契约
   - HMI 轨迹图层仅使用离散点元素进行显示，不引入连线元素作为主轨迹表达。
   - 在 rect_diag.dxf 上，视觉上可区分为离散点列，而非连续实线带。
2. 小尾巴消除判据
   - 在 rect_diag.dxf 的角点/交点附近，不出现可见的反向短回折“尾巴”。
   - 轨迹点序列的 ABA 回折计数满足目标阈值（默认目标：0，容差按实现说明固化）。
3. 几何一致性
   - 预览轨迹外包框与输入几何外包框一致（考虑配置偏移后的同一坐标系），不出现额外越界尖刺。
4. 数据链路一致性
   - dxf.preview.snapshot 返回的 trajectory_polyline 与 HMI 渲染点集一一对应（无隐式二次平滑导致形变）。
5. 门禁不回归
   - preview confirm 与 snapshot hash 校验行为保持不变；未确认时禁止启动。
6. 回归通过
   - 新增/更新的单元测试与最小联调验证通过，至少覆盖 rect_diag.dxf 场景。

## 6. 风险与前置依赖

1. 必须在真实 runtime 链路验证；mock 数据会掩盖或伪造几何问题。
2. 运动插补与预览抽样复用同一轨迹时，角点速度连续性策略可能带来视觉过冲，需要明确“预览用途优先级”。
3. max_polyline_points 截断可能引入抽样别名，导致局部伪影，需要同步评估抽样策略。
4. 点间距与点半径配置不当会造成“看起来像线”，即使算法正确，需同时约束渲染参数。

## 7. 明确不做项

1. 不在本阶段重构 motion/interpolation 全栈算法。
2. 不扩展新的协议命令面，仅在现有 snapshot 链路内修复。
3. 不处理与本缺陷无关的页面样式和文案优化。

## 8. 关键澄清问题（最多 5 个）

1. 点状预览的判定口径是否接受“离散点优先，可见点间隔”这一标准，而不是仅看是否由 circle 绘制？
2. 交点“小尾巴”是否按“默认零容忍（ABA 回折计数目标为 0）”执行？
3. 预览应优先表达“几何真实形状”还是“执行动态轨迹（含过渡平滑）”？
4. 允许的几何偏差阈值是否采用毫米级固定阈值（建议在实现阶段固化）？
5. 本次验收样例是否以 rect_diag.dxf 为强制基线，并要求可重复自动回归？

## 9. 推荐默认答案（用于直接进入实现）

1. 采用“离散点可见间隔”作为判定口径，不以元素类型（circle）替代验收。
2. 小尾巴按零容忍执行，ABA 回折计数目标为 0。
3. 预览优先表达几何真实形状，不引入执行态平滑过冲。
4. 几何偏差阈值采用固定毫米级阈值，并在实现文档中显式写死（不依赖主观目测）。
5. rect_diag.dxf 作为强制回归样例，纳入自动化回归。
