# Process Model

本目录默认属于历史文档区。

它承接任务执行过程中的中间产物与追溯材料，而不是当前工作区的正式真值入口。

## 子目录

- `plans/`：任务规划总结 —— 仅保留 wave 级汇总收口文件（批量操作计划已移入 `_archive/process-model/plans/`）
- `reviews/`：评审与验收记录 —— 仅保留最终版/汇总版（多版本时间戳评审已移入 `_archive/process-model/reviews/`）
- `retro/`：复盘材料

## 追溯入口

1. 需要看"这次任务怎么计划和分批推进"，读 `plans/`
2. 需要看"这次任务如何审查、验收或 closeout"，读 `reviews/`
3. 需要看"事后复盘和阶段经验"，读 `retro/`
4. 如果只是理解当前正式规则、当前架构或当前验证口径，应直接回到正式文档区，不要先读本目录

## 容量警戒

当前 `plans/` + `reviews/` 文件数应控制在 30 以内。超出时应主动将已完成的批次计划归档至 `_archive/process-model/`。

## 使用规则

1. `Codex` 不应默认把本目录内容视为当前正式规范
2. 当正式文档无法解释某个历史决策或实施过程时，才回看本目录
3. 如果本目录中的结论仍然有效并被持续依赖，应将该结论上收至正式文档区
