# NOISSUE UI Plan Check

- 日期：2026-03-21
- 分支：`feat/hmi/NOISSUE-dispense-trajectory-preview-v2`
- 依据：
  - `docs/process-model/plans/NOISSUE-scope.md`
  - `docs/process-model/plans/NOISSUE-arch-plan.md`
  - `apps/hmi-app/src/hmi_client/ui/main_window.py`

## 1. 审查对象

本次审查仅覆盖与“点状几何轨迹预览 + 执行前确认门禁”直接相关的 UI：

1. 生产页左侧“作业准备”卡片（文件、信息、刷新预览）
2. 右侧预览画布区域（WebEngine 预览 / 降级提示）
3. 启动前预检与确认弹窗流程
4. 联机模式状态栏（Offline/Online、TCP、硬件）

## 2. 状态覆盖检查（5 类）

### 2.1 生产页 - 作业准备卡片（`_create_production_tab`）

- 初始/加载：部分覆盖  
  - 有初始文案 `段数: - | 长度: - | 预估: - | 预览: 待生成`（`main_window.py:561`）。
  - 有“生成中”状态文本映射（`main_window.py:1774-1783`）。
  - 但生成逻辑同步阻塞主线程（`main_window.py:1955-2003` + `preview_client.py:73-82`），用户体感为界面卡顿，加载态不可感知。
- 正常：覆盖  
  - 预览成功后更新状态条并提示“启动前需确认”（`main_window.py:2015-2019`）。
- 空状态：覆盖不足  
  - 未加载 DXF 时点击“刷新预览”直接返回，无提示（`main_window.py:1936-1937`）。
- 错误：覆盖  
  - 工件缺失、组件不可用、快照失败、渲染失败均有状态栏提示（`main_window.py:1938-1947,1961-1966,2003-2013`）。
- 权限/网络异常：部分覆盖  
  - Offline 模式有统一门禁提示（`main_window.py:1301-1304`）。
  - 轮询异常仅记录日志，无可见降级提示（`main_window.py:2450-2451`）。

### 2.2 预览画布（`_dxf_view`）

- 初始/加载：部分覆盖  
  - WebEngine 缺失时有静态提示（`main_window.py:718-723`）。
- 正常：当前实现与范围基线不一致  
  - 预览来源是本地 DXF -> `engineering-data` HTML（`main_window.py:1998-2002`，`preview_client.py:48-62`），不是“主链路快照点状几何数据”。
- 空状态：覆盖不足  
  - 首次进入页面时无“请加载 DXF/请生成预览”的显式空态说明，仅依赖默认空白或历史内容。
- 错误：覆盖  
  - 渲染失败会直接写入红色错误 HTML（`main_window.py:2007-2011`）。
- 权限/网络异常：覆盖不足  
  - 网络中断后画布不展示“数据过期/连接异常”标识，用户无法判断当前预览是否仍可信。

### 2.3 启动前预检与确认（`_check_production_preconditions`）

- 初始/加载：覆盖  
  - 未连接、未初始化、无计划等前置条件均有拦截提示（`main_window.py:2052-2069`）。
- 正常：覆盖  
  - 通过门禁与哈希校验后允许启动（`main_window.py:2078-2094`）。
- 空状态：覆盖  
  - 未生成计划时提示“预览计划未生成”（`main_window.py:2067-2069`）。
- 错误：覆盖  
  - 预览失败、过期、哈希不一致均有阻断文案（`main_window.py:1785-1800`）。
- 权限/网络异常：部分覆盖  
  - Offline 门禁覆盖；但运行期网络抖动时，状态可能停留在旧值且缺少显式“状态不可信”提示（`main_window.py:2320-2451`）。

## 3. 响应式与键盘可达性

### 3.1 响应式

- 结论：当前为固定工位 UI，不是响应式布局。  
  - 主窗口最小尺寸固定（`main_window.py:244-247`）。  
  - 生产页侧边栏固定宽度 `400`（`main_window.py:514-516`）。
- 影响：若切换到小分辨率/窄屏，信息层级可能压缩；不适合作为移动端方案。

### 3.2 键盘可达性

- 结论：基础可用但未体系化。  
  - 未发现统一 `setTabOrder`/快捷键编排。  
  - 关键操作按钮可通过默认焦点导航访问，但点动按钮显式禁用焦点（`main_window.py:110`）。
- 影响：在键盘优先或无鼠标场景下，操作路径可预测性不足。

## 4. 必须修复项（进入实现前）

1. 预览数据链与冻结边界不一致（P0）  
组件：生产页预览画布、预览生成流程  
证据：`main_window.py:1998-2002`、`preview_client.py:48-62`、`main_window.py:1840-1890`（点状快照渲染函数存在但未接入）  
结论：当前渲染来自本地 DXF HTML，不满足“主链路快照点状几何路径”目标。

2. 空状态操作无反馈（P0）  
组件：刷新预览按钮  
证据：`main_window.py:1936-1937`  
结论：未加载 DXF 时点击刷新静默返回，用户无法判断操作是否生效。

3. 加载态不可感知且可能导致 UI 假死（P1）  
组件：预览生成流程  
证据：`main_window.py:1955-2003`、`preview_client.py:73-82`  
结论：同步外部进程调用阻塞 UI 线程，`GENERATING` 状态缺少可交互反馈。

4. 运行期网络/状态异常的可见性不足（P1）  
组件：全局状态区与预览可信性提示  
证据：`main_window.py:2450-2451`  
结论：状态轮询异常仅写日志，界面无“状态不可用/数据可能过期”提示。

## 5. 可后置项（不阻断本轮）

1. 键盘导航与快捷键体系化（P2）  
组件：生产页整体操作路径  
理由：不阻断点状预览主目标，但影响可达性与操作效率。

2. 响应式/小屏适配（P2）  
组件：生产页布局（固定最小尺寸 + 固定侧栏）  
理由：当前工位型 HMI 可接受，后续若扩展到不同分辨率需专项改造。

3. 预览信息层级增强（P2）  
组件：预览区空态与错误态文案  
理由：现有状态栏提示可用，但持久化信息层级不足，后续可优化。

## 6. 审查结论

当前 UI 框架已具备预览门禁基础能力，但尚未达到本轮冻结目标。  
进入实现阶段前，必须先解决第 4 节四项问题，尤其是“点状快照数据链闭环”与“空状态可感知反馈”。
