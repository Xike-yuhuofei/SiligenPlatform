## 6. 模块结论
- 宣称职责：`ProcessPath -> MotionPlan(MotionTrajectory)` 的运动规划 owner，负责时间/轨迹/CMP 规划与插补算法，runtime/control 已迁出。[modules/motion-planning/README.md](/D:/Projects/SiligenSuite/modules/motion-planning/README.md#L5) [modules/motion-planning/domain/motion/README.md](/D:/Projects/SiligenSuite/modules/motion-planning/domain/motion/README.md#L3)
- 实际职责：运动规划 + 插补程序生成 + CMP/trigger 规划 + 公开执行兼容层 + 持有一个实际执行控制端口。
- 是否职责单一：否。
- 是否边界清晰：否。
- 是否被侵入：是。workflow/runtime-execution 兼容头与外部直连 domain header 都在侵入它的 public surface。
- 是否侵入别人：是。执行插补控制语义侵入了 `runtime-execution`，CMP/trigger 语义侵入了 `dispense-packaging`。
- 是否适合作为稳定业务模块继续演进：当前不适合；先收口 public surface 和 owner 再扩展。
- 最终评级：明显偏移。
